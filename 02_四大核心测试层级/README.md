# 02_四大核心测试层级

## 模块简介

本模块介绍软件测试的四大核心测试层级，从底层的单元测试到顶层的验收测试，形成经典的"测试金字塔"模型。每个层级对应不同的测试目标、测试粒度和测试方法，是理解软件测试工程体系的战略框架。

---

## 目录结构

```
02_四大核心测试层级/
├── README.md           # 本文件
├── 01_单元测试.md       # 函数/方法/类级别的正确性验证，测试金字塔的基座
├── 02_集成测试.md       # 模块间接口、交互、数据传递的验证
├── 03_系统测试.md       # 完整系统的端到端功能与非功能验证
└── 04_验收测试.md       # 用户/客户视角的业务接受度验证，上线最终审批
```

---

## 测试金字塔

```
          /\
         /  \          验收测试 (少量，端到端业务验证)
        /    \
       /------\
      /        \       系统测试 (中量，完整系统验证)
     /          \
    /------------\
   /              \   集成测试 (中量，模块间协作验证)
  /                \
 /__________________\  单元测试 (大量，代码级快速反馈)
```

---

## 学习路线图

```
第1步: 01_单元测试        理解最小的测试单元和代码级验证方法
  │
第2步: 02_集成测试        掌握模块间接口和交互的测试策略
  │
第3步: 03_系统测试        学习端到端的全系统测试方法
  │
第4步: 04_验收测试        理解用户视角的业务验收标准和流程
```

---

## 学习建议

1. 按照测试金字塔自底向上的顺序学习，每个层级的测试技术是上一层级的基础
2. 理解各层级的测试目标差异：单元测试验证"做对了"，集成测试验证"能协作"，系统测试验证"符合需求"，验收测试验证"可交付"
3. 注意各层级对应的测试方法：单元测试偏白盒，集成测试偏灰盒，系统测试和验收测试偏黑盒
4. 结合 Module 00（测试方法核心知识库）交叉学习，理解测试方法与测试层级的对应关系
5. 建议学习周期：2-3周
   - 第1周：01-02（单元+集成）
   - 第2周：03-04（系统+验收）
