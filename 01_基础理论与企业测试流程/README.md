# 01_基础理论与企业测试流程

## 模块简介

本模块是测试工程学习知识库的入门模块，涵盖软件测试的基础理论、企业级测试流程、主流测试模型、测试方法论、多环境管理、测试左移右移、质量模型和常见问题解答，为后续各模块的学习奠定理论基础。

---

## 目录结构

```
01_基础理论与企业测试流程/
├── README.md              # 本文件
├── 01_测试基础理论.md      # 软件测试定义、分类、原则与核心概念
├── 02_企业测试流程.md      # 需求分析、测试计划、设计、执行和评估五大阶段
├── 03_主流测试模型.md      # V模型、W模型、敏捷测试、DevOps测试等模型对比
├── 04_测试方法论.md        # 基于风险/需求/模型/经验的测试方法论
├── 05_多环境管理.md        # 开发/测试/预发布/生产环境的配置与管理
├── 06_测试左移.md         # 需求评审、设计评审、单元测试等早期介入策略
├── 07_测试右移.md         # 灰度发布、线上监控、混沌工程等生产验证
├── 08_质量模型.md         # ISO 25010质量特性与质量度量框架
└── 09_常见问题.md         # 基础理论与流程相关的常见问题解答
```

## 学习路线图

```
第1步: 01_测试基础理论      建立软件测试的核心概念体系
  │
第2步: 02_企业测试流程      理解企业级测试活动的完整生命周期
  │
第3步: 03_主流测试模型      了解不同开发模型下测试活动的组织方式
  │
第4步: 04_测试方法论        掌握指导测试设计的系统性方法论
  │
第5步: 05_多环境管理        学会管理和区分不同用途的运行环境
  │
第6步: 06_测试左移          学习将测试活动提前到开发早期
  │
第7步: 07_测试右移          学习将测试活动延伸到生产环境
  │
第8步: 08_质量模型          理解软件质量的标准化度量框架
  │
第9步: 09_常见问题          查阅常见疑问和实战经验解答
```

## 学习建议

1. 按目录顺序循序渐进学习，01-04 是理论基础，05-08 是实践延伸，09 是随时参考
2. 学习中结合 Module 00（测试方法核心知识库）交叉理解测试方法的应用场景
3. 重点关注 02（企业测试流程）和 06-07（测试左移右移），二者是现代测试工程化的核心理念
4. 参考本模块 09_常见问题.md 快速解决学习中的疑惑
5. 建议学习周期：1-2周
