# 00_测试方法核心知识库

## 模块简介

本模块是测试工程学习知识库的核心基础，涵盖软件测试的三大测试方法分类：黑盒测试、白盒测试和灰盒测试。每种方法都包含完整的测试设计技术汇总，是学习后续各模块测试技术的理论基石。


## 三大测试方法

| 分类 | 核心理念 | 适用场景 |
|------|---------|---------|
| **黑盒测试** | 不关注内部结构，仅通过输入/输出验证功能 | 功能测试、系统测试、验收测试 |
| **白盒测试** | 基于代码内部逻辑和结构进行测试 | 单元测试、代码审查、安全审计 |
| **灰盒测试** | 结合外部行为验证与内部结构了解 | 接口测试、集成测试、数据库测试 |

---

## 学习建议

1. **第1步：黑盒测试** - 最常用、最基础的测试方法，所有测试工程师必须熟练掌握
2. **第2步：灰盒测试** - 在现代接口测试和集成测试中应用广泛，是测试工程师的核心竞争力
3. **第3步：白盒测试** - 理解代码逻辑，提升测试设计的精准度，是向测试开发进阶的基础
4. 每种方法学习后，建议结合后续模块（01-04）中的实战场景加深理解
5. 建议学习周期：1-2周
