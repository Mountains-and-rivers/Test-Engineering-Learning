# 05_缺陷管理&线上问题体系

## 模块简介

本模块涵盖软件测试中的缺陷管理全流程与线上问题应对体系，从Bug的等级判定、规范提交、案例复盘，到环境排查、事故复盘、线上巡检和应急处理，形成完整的"发现问题→定位问题→修复问题→预防问题"闭环。

---

## 学习路线图

```
第1步: 01_缺陷等级规范         理解Bug如何分级（严重度 vs 优先级）
  │
第2步: 02_缺陷提交规范         掌握如何写出高质量的Bug报告
  │
第3步: 03_Bug案例复盘         学习从Bug中提取最大价值的方法论
  │
第4步: 04_环境排查思路         掌握"我的环境没问题"的排查方法
  │                              （本步可并行学习）
  ▼
第5步: 05_事故复盘体系         建立系统化的事故应对和改进机制
  │
第6步: 06_线上巡检             构建自动化巡检体系，在用户之前发现问题
  │
第7步: 07_应急处理             掌握线上事故应急响应和处理能力
```

---

## 学习建议

1. **按顺序学习01-03**：先掌握Bug的基础规范（等级+提交），再学习如何复盘和沉淀价值
2. **04可独立学习**：环境排查是日常高频技能，可随时查阅
3. **05-07建议连续学习**：事故复盘、线上巡检、应急处理三者紧密关联，形成线上质量保障的完整闭环
4. **结合实战项目进行实践**：将规范模板应用到实际项目中
5. **参考09_常见问题汇总快速解决疑惑**
6. **建议学习周期：2-3周**
   - 第1周：01-03（缺陷管理基础）
   - 第2周：04-05（环境排查+事故体系）
   - 第3周：06-07（巡检+应急）

## 核心概念速览

| 概念 | 一句话解释 | 参考文档 |
|------|-----------|----------|
| Severity vs Priority | 严重度是Bug多严重（不变），优先级是应该多快修（可变） | 01 |
| Given-When-Then | 标准化的Bug复现步骤编写格式 | 02 |
| 5-Why分析法 | 连续问5个"为什么"找到根本原因 | 03 |
| 分层排查法 | 从网络层→服务层→应用层逐层排查环境问题 | 04 |
| Blameless Postmortem | 不追责、聚焦系统改进的事故复盘文化 | 05 |
| 四维巡检模型 | 业务/服务/资源/安全四个维度的线上巡检 | 06 |
| 先止损后排查 | 事故处理第一原则：先止血再找病因 | 07 |
