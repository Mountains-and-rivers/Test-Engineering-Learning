## UI自动化概述

UI自动化测试是验证用户界面功能正确性的关键手段。选择正确的工具，对测试效率、维护成本和覆盖率至关重要。

### 自动化测试金字塔

```
          ╱  E2E 测试 ╲
         ╱  (少量、慢、脆弱)  ╲
        ╱───────────────────────╲
       ╱   集成/API 测试          ╲
      ╱   (中等量、中等速度)        ╲
     ╱─────────────────────────────╲
    ╱       单元测试                  ╲
   ╱       (大量、快、稳定)             ╲
  ╱───────────────────────────────────╲
```

> **注意：** 测试金字塔是经典模型，至今仍有参考价值。随着微服务和前端框架的演进，社区提出了多个现代变体：
> - **Testing Trophy（测试奖杯）** 由 Kent C. Dodds 提出，强调前端测试中"集成测试"应占最大比重（Static > Unit > Integration > E2E）
> - **Test Honeycomb（测试蜂巢）** 由 Spotify 推广，适用于微服务架构，强调集成测试在服务边界上的核心地位
> 
> 选型时应根据项目架构（单体/微服务/前端重型）选择最适合的模型，而非盲目套用金字塔。

## Selenium WebDriver 详解

### 2.1 架构原理

Selenium WebDriver 通过浏览器驱动(Browser Driver)与浏览器通信：

```
[测试代码] → [WebDriver API] → [Browser Driver] → [Browser]
             (语言绑定)          (ChromeDriver/      (Chrome/
                                  GeckoDriver/        Firefox/
                                  EdgeDriver)         Edge)
```

### 2.2 环境搭建

**Python + Selenium 环境：**
```bash
pip install selenium
pip install webdriver-manager  # 自动管理驱动版本
```

**Java + Selenium 环境 (Maven)：**
```xml
<dependency>
    <groupId>org.seleniumhq.selenium</groupId>
    <artifactId>selenium-java</artifactId>
    <version>4.18.1</version>
</dependency>
```

### 2.4 Page Object Model (POM)

```python
# pages/base_page.py
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

class BasePage:
    def __init__(self, driver):
        self.driver = driver
        self.wait = WebDriverWait(driver, 10)

    def find(self, by, locator):
        return self.wait.until(EC.presence_of_element_located((by, locator)))

    def click(self, by, locator):
        self.wait.until(EC.element_to_be_clickable((by, locator))).click()

    def type(self, by, locator, text):
        el = self.find(by, locator)
        el.clear()
        el.send_keys(text)

# pages/login_page.py
from selenium.webdriver.common.by import By
from pages.base_page import BasePage

class LoginPage(BasePage):
    USERNAME_INPUT = (By.ID, "username")
    PASSWORD_INPUT = (By.ID, "password")
    LOGIN_BUTTON = (By.ID, "login-btn")
    ERROR_MESSAGE = (By.CLASS_NAME, "error-msg")

    def login(self, username, password):
        self.type(*self.USERNAME_INPUT, username)
        self.type(*self.PASSWORD_INPUT, password)
        self.click(*self.LOGIN_BUTTON)

    def get_error_message(self):
        return self.find(*self.ERROR_MESSAGE).text

# tests/test_login.py
import pytest
from selenium import webdriver
from pages.login_page import LoginPage

class TestLogin:
    @pytest.fixture
    def driver(self):
        driver = webdriver.Chrome()
        driver.get("https://example.com/login")
        yield driver
        driver.quit()

    def test_login_success(self, driver):
        login_page = LoginPage(driver)
        login_page.login("admin", "password123")
        assert "欢迎" in driver.page_source

    def test_login_failed(self, driver):
        login_page = LoginPage(driver)
        login_page.login("admin", "wrong_password")
        assert "密码错误" in login_page.get_error_message()
```

## Playwright 新一代自动化框架

### 3.1 概述

Playwright 由 Microsoft 开发，是 Selenium 的强力竞争者。

**核心优势：**
- 自动等待（Auto-wait）- 不需要显式等待
- 多浏览器支持（Chromium, Firefox, WebKit）
- 移动端模拟
- 网络拦截与Mock
- 自动录制与代码生成
- 并行测试执行

**安装：**
```bash
pip install playwright
playwright install  # 安装浏览器二进制文件
```

### 3.2 基础使用

```python
from playwright.sync_api import sync_playwright, expect

with sync_playwright() as p:
    browser = p.chromium.launch(headless=False)
    context = browser.new_context(
        viewport={'width': 1920, 'height': 1080},
        locale='zh-CN'
    )
    page = context.new_page()

    # 导航
    page.goto("https://example.com/login")

    # 自动等待 + 输入（不需要等待元素出现）
    page.fill("#username", "admin")
    page.fill("#password", "password123")
    page.click("#login-btn")

    # 断言
    expect(page.locator(".welcome-text")).to_contain_text("欢迎")

    # 截屏
    page.screenshot(path="screenshots/login-success.png", full_page=True)

    browser.close()
```

**异步API：**
```python
import asyncio
from playwright.async_api import async_playwright

async def main():
    async with async_playwright() as p:
        browser = await p.chromium.launch()
        page = await browser.new_page()
        await page.goto("https://example.com")
        title = await page.title()
        print(title)
        await browser.close()

asyncio.run(main())
```

### 3.3 Playwright 高级特性

```python
# 1. 网络拦截和Mock
page.route("**/api/users", lambda route: route.fulfill(
    status=200,
    content_type="application/json",
    body='{"users": [{"id": 1, "name": "Mock用户"}]}'
))

# 2. 监听网络请求
page.on("request", lambda request: print(f"请求: {request.url}"))
page.on("response", lambda response: print(f"响应: {response.status} {response.url}"))

# 3. 文件上传
page.set_input_files("#file-upload", "/path/to/file.pdf")

# 4. 文件下载
with page.expect_download() as download_info:
    page.click("#download-btn")
download = download_info.value
download.save_as("/path/to/save/file.pdf")

# 5. 多页面/多标签处理
with context.expect_page() as new_page_info:
    page.click('a[target="_blank"]')
new_page = new_page_info.value

# 6. 移动端模拟
iphone = p.devices['iPhone 12']
context = browser.new_context(**iphone)
page = context.new_page()

# 7. 地理位置模拟
context = browser.new_context(
    geolocation={"longitude": 116.4074, "latitude": 39.9042},
    permissions=["geolocation"]
)

# 8. 认证状态复用（保存登录状态）
# 保存
context.storage_state(path="auth.json")
# 复用
context = browser.new_context(storage_state="auth.json")

# 9. API Testing（直接发HTTP请求）
response = page.request.post("https://api.example.com/users", data={
    "name": "张三",
    "email": "zhangsan@example.com"
})
assert response.ok

# 10. 代码生成器
# playwright codegen https://example.com
```

### 3.4 Playwright vs Selenium 对比

| 维度 | Playwright | Selenium |
|------|-----------|----------|
| **浏览器支持** | Chromium, Firefox, WebKit | Chrome, Firefox, Edge, Safari, IE |
| **语言支持** | JS/TS, Python, Java, .NET | Java, Python, C#, Ruby, JS, Kotlin |
| **自动等待** | 内置 | 需手动实现 |
| **网络拦截** | 原生支持 | 需要通过代理 |
| **移动端模拟** | 原生支持 | 需要Appium配合 |
| **执行速度** | 较快 | 较慢 |
| **社区生态** | 快速增长 | 成熟庞大 |
| **调试工具** | Trace Viewer, Inspector | 依赖第三方 |
| **API Testing** | 内置request对象 | 不支持 |
| **IE支持** | 不支持 | 支持 |
| **学习曲线** | 低 | 中 |

## Cypress 前端自动化

### 4.1 概述

Cypress 是专为现代Web应用设计的前端端到端测试框架。

**特点：**
- 与浏览器在同一运行循环中执行（更快、更稳定）
- 自动截图和录像
- 时间旅行调试（Time Travel）
- 实时重载
- 丰富的断言库

**安装：**
```bash
npm install cypress --save-dev
npx cypress open
```

### 4.2 基础测试编写

```javascript
// cypress/e2e/login.cy.js
describe('登录功能测试', () => {
  beforeEach(() => {
    cy.visit('/login');
  });

  it('正常登录', () => {
    cy.get('[data-cy=username]').type('admin');
    cy.get('[data-cy=password]').type('password123');
    cy.get('[data-cy=login-btn]').click();
    cy.url().should('include', '/dashboard');
    cy.contains('欢迎回来').should('be.visible');
  });

  it('登录失败 - 错误密码', () => {
    cy.get('[data-cy=username]').type('admin');
    cy.get('[data-cy=password]').type('wrong');
    cy.get('[data-cy=login-btn]').click();
    cy.get('.error-message').should('contain', '密码错误');
  });
});

// API Mock示例
describe('Mock数据测试', () => {
  it('使用Mock数据渲染页面', () => {
    cy.intercept('GET', '/api/users', {
      statusCode: 200,
      body: {
        users: [
          { id: 1, name: '张三' },
          { id: 2, name: '李四' }
        ]
      }
    }).as('getUsers');

    cy.visit('/users');
    cy.wait('@getUsers');
    cy.get('.user-list li').should('have.length', 2);
  });

  it('拦截请求并修改响应', () => {
    cy.intercept('POST', '/api/login', (req) => {
      req.reply({
        statusCode: 200,
        body: { token: 'mock-token-123', role: 'admin' }
      });
    }).as('login');

    cy.get('[data-cy=login-btn]').click();
    cy.wait('@login');
  });
});
```

## Appium 移动端自动化


### 技巧1: Selenium 元素等待策略

```python
# 不推荐：使用 time.sleep()
# time.sleep(3)  # 浪费时间，且不稳定

# 推荐：显式等待 + 自定义条件
from selenium.webdriver.support import expected_conditions as EC

class element_has_no_css_class:
    """自定义等待条件：元素不包含某个CSS类"""
    def __init__(self, locator, css_class):
        self.locator = locator
        self.css_class = css_class

    def __call__(self, driver):
        element = driver.find_element(*self.locator)
        if self.css_class not in element.get_attribute("class"):
            return element
        else:
            return False

# 使用
wait.until(element_has_no_css_class((By.ID, "spinner"), "loading"))
```

### 技巧5: Robust Locator 策略

优先级从高到低：

1. **data-testid / data-cy (最佳)**
```python
# Python
driver.find_element(By.CSS_SELECTOR, '[data-testid="submit-btn"]')

# Playwright
page.locator('[data-testid="submit-btn"]')

# Cypress
cy.get('[data-cy="submit-btn"]')
```

2. **语义化选择器**
```python
page.get_by_role("button", name="提交")
page.get_by_label("用户名")
page.get_by_placeholder("请输入密码")
page.get_by_text("忘记密码？")
```

3. **基于结构的CSS选择器（避免使用脆弱的选择器）**
```css
/* 好：基于语义 */
input[name="email"]

/* 差：脆弱（依赖布局结构） */
div:nth-child(3) > div > input:nth-child(1)
```

4. **XPATH（最后手段）**
```xpath
//button[contains(text(), '提交')]
//input[@placeholder='请输入用户名']
```

## 总结

选择自动化工具的核心考量因素：

1. **应用类型** - Web / 移动端 / API / 桌面应用的组合
2. **团队技能** - 编程语言偏好和技术栈
3. **浏览器覆盖** - 是否需要IE支持
4. **执行速度** - 快速反馈 vs 全面覆盖
5. **维护成本** - 长期看，易维护性比初始开发速度更重要

对于新项目，首选推荐组合：
- Web: **Playwright** (现代Web应用首选)
- API: **Playwright内置** 或 **Requests/Pytest**
- 移动端: **Appium**
- 游戏: **Airtest**
