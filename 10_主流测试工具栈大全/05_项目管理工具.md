
## Jira 缺陷管理与敏捷看板

### 2.1 Jira 核心概念

Jira 是 Atlassian 出品的业界标准项目管理工具。

**核心概念层级：**
```
Site (站点)
└── Project (项目)
    ├── Issue Type (问题类型)
    │   ├── Epic (史诗 - 大型功能)
    │   ├── Story (用户故事)
    │   ├── Task (任务)
    │   ├── Bug (缺陷)
    │   └── Sub-task (子任务)
    ├── Workflow (工作流)
    │   └── Transition (状态转换)
    │       └── Status (状态)
    ├── Board (看板)
    │   ├── Scrum Board (Scrum看板)
    │   └── Kanban Board (看板)
    ├── Sprint (迭代)
    ├── Version/Release (版本)
    └── Component (组件/模块)
```

### 2.2 测试缺陷管理最佳实践

**Bug Issue Template：**
```
【标题】Brief and precise description（简洁精准的描述）

【环境 (Environment)】
- 操作系统: Windows 10 / macOS 13 / Linux
- 浏览器: Chrome 120.0 / Firefox 118
- 测试环境: Staging / Production
- 应用版本: v2.3.1

【前置条件 (Precondition)】
1. 用户已登录
2. 用户拥有管理员权限
3. 已有至少一条历史订单

【复现步骤 (Steps to Reproduce)】
1. 进入「订单管理」页面
2. 点击「批量导出」按钮
3. 选择日期范围 2024-01-01 至 2024-01-31
4. 点击「导出Excel」

【实际结果 (Actual Result)】
导出文件仅包含10条记录，但订单管理页面显示有15条记录

【期望结果 (Expected Result)】
导出文件应包含日期范围内的全部15条记录

【复现概率】
5/5 (100%)

【附件 (Attachment)】
- 截图: export_issue.png
- 录屏: export_bug.mp4
- 日志: application.log

【严重程度 (Severity)】
Critical / Major / Minor / Trivial

【优先级 (Priority)】
Blocker / Highest / High / Medium / Low
```

### 2.4 Jira JQL (Jira Query Language)

JQL 是 Jira 强大的搜索语言，掌握JQL是高效使用Jira的关键：

```sql
-- 基础搜索
project = "MyProject" AND type = Bug
status = "In Progress" AND assignee = currentUser()
created >= -7d
updated <= -1d

-- 版本相关
fixVersion = "v2.4.0"
affectedVersion = "v2.3.0"
affectedVersion in ("v2.3.0", "v2.3.1")

-- 测试相关高级JQL
-- 本周创建的Bug
project = TEST AND issuetype = Bug AND created >= startOfWeek()

-- 高优先级未解决的Bug
priority in (Blocker, Highest, High) AND status not in (Resolved, Closed)

-- 分配给我的待验证Bug
assignee = currentUser() AND status = "Resolved" AND labels = "待验证"

-- 临近截止日期的任务
duedate < endOfDay(3) AND status != Closed

-- 回归失败的Bug
labels = "regression" AND status changed to Reopened after -7d

-- 按模块统计Bug
project = TEST AND issuetype = Bug AND component = "支付模块"

-- 按严重程度排序
project = TEST AND issuetype = Bug ORDER BY priority DESC, created DESC
```

### 2.5 Jira 与测试工具集成

**Jira + Xray (测试管理插件)：**
```
Xray 核心概念:
- Test: 测试用例
- Test Set: 测试用例集
- Test Plan: 测试计划
- Test Execution: 测试执行记录

工作流：
1. 创建Test（关联到Story/Requirement）
2. 组织Test Set
3. 创建Test Plan（关联测试环境、版本）
```

```
Zephyr 特点：
- 测试循环 (Test Cycle)
- 批量执行
- 测试仪表板
- 自动关联缺陷
```

## 禅道 国产项目管理

### 3.1 概述

禅道 (ZenTao) 是国内最流行的开源项目管理软件，内置产品管理、项目管理、测试管理、文档管理等功能。

**部署方式：**
```bash
# Docker 一键部署 (推荐)
docker run -d -p 80:80 \
  -v /data/zbox:/opt/zbox/data \
  --name zentao \
  easysoft/zentao:latest

# 访问 http://localhost
# 默认账号: admin / 123456

# Linux 一键安装包
wget https://www.zentao.net/dl/ZenTaoPMS.18.10.zbox_64.tar.gz
tar -xzf ZenTaoPMS.18.10.zbox_64.tar.gz
/opt/zbox/zbox start
```

### 3.2 模块功能

| 模块 | 功能 | 使用者 |
|------|------|--------|
| **产品** | 需求管理、产品路线图 | 产品经理 |
| **项目** | 任务分解、迭代管理、看板 | 项目经理/开发 |
| **测试** | 用例管理、测试单、Bug | 测试团队 |
| **文档** | 产品文档、技术文档 | 全体 |
| **统计** | 仪表盘、各类报表 | 管理层 |
| **组织** | 用户、部门、权限 | 管理员 |

### 3.3 测试管理流程

```
禅道测试流程：
1. 创建产品 → 维护模块 → 录入需求
2. 创建项目 → 关联需求 → 分解任务
3. 编写测试用例 → 关联需求
4. 提交测试版本 → 创建测试单
5. 执行测试 → 记录结果
6. 发现Bug → 提交到禅道Bug模块
7. Bug修复 → 回归验证
8. 测试报告
```

**Bug字段规范：**
```
[标题] 格式： [模块] 简洁的问题描述
例如: [登录] 验证码刷新后session未更新导致验证失败

[严重程度]
- 1级: 系统崩溃、数据丢失、核心功能不可用
- 2级: 主要功能异常、严重影响使用
- 3级: 次要功能异常、一般性缺陷
- 4级: 界面展示问题、提示文字错误

[优先级]
- 1: 立即修复(阻塞测试)
- 2: 本迭代修复
- 3: 下迭代修复
- 4: 暂不处理

[类型]
- 代码错误
- 设计缺陷
- 界面优化
- 配置问题
- 性能问题
- 安全漏洞
```

### 3.4 禅道API

```python
# 禅道API调用示例
import requests
import hashlib

BASE_URL = "http://zentao.example.com/api.php/v1"
TOKEN = ""

# 获取Token
def get_token(username, password):
    # 禅道需要先获取session
    session = requests.Session()
    login_url = f"{BASE_URL}/tokens"
    data = {"account": username, "password": password}
    response = session.post(login_url, json=data)
    return response.json().get('token')

# 创建Bug
def create_bug(product_id, title, severity, steps):
    headers = {"Token": TOKEN}
    bug_data = {
        "product": product_id,
        "title": title,
        "severity": severity,
        "steps": steps,
        "type": "codeerror"
    }
    resp = requests.post(f"{BASE_URL}/bugs", json=bug_data, headers=headers)
    return resp.json()

# 获取产品下的Bug列表
def get_bugs(product_id):
    headers = {"Token": TOKEN}
    resp = requests.get(f"{BASE_URL}/products/{product_id}/bugs", headers=headers)
    return resp.json()
```

TAPD (Tencent Agile Product Development) 是腾讯出品的敏捷研发协作平台，提供了需求、缺陷、任务、测试、发布等全方位管理能力。

```
TAPD 功能模块：
├── 需求 (Story)
│   ├── 需求池
│   ├── 需求优先级 (MoSCoW)
│   ├── 需求评审

│   ├── 迭代规划

```

### 4.3 TAPD 测试管理

```
TAPD 测试计划流程：
1. 创建测试计划 → 关联迭代
2. 从用例库中选择/导入用例
3. 分配测试任务给测试人员
4. 执行测试 → 记录通过/失败/阻塞
5. 失败用例 → 一键创建缺陷（自动关联用例和需求）
6. 测试报告生成
```

**TAPD API 示例：**
```python
import requests

# TAPD API (使用 Basic Auth)
api_user = "your_api_user"
api_password = "your_api_password"
workspace_id = "12345678"

def create_bug(title, description, priority="高"):
    url = "http://apiv2.api.tapd.cn/bugs"
    auth = (api_user, api_password)
    data = {
        "workspace_id": workspace_id,
        "title": title,
        "description": description,
        "priority_label": priority,
    }
    response = requests.post(url, auth=auth, data=data)
    return response.json()

def get_bugs(status="new"):
    url = "http://apiv2.api.tapd.cn/bugs"
    auth = (api_user, api_password)
    params = {
        "workspace_id": workspace_id,
        "status": status,
        "limit": 100
    }
    response = requests.get(url, auth=auth, params=params)
    return response.json()
```

## TestLink/TestRail 测试用例管理

### 5.1 TestLink

TestLink 是开源的测试管理工具，专注于测试用例管理和执行跟踪。

**Docker部署：**
```bash
docker run -d --name testlink \
  -p 8080:80 \
  -e TESTLINK_DATABASE_HOST=db \
  -e TESTLINK_DATABASE_NAME=testlink \
  -e TESTLINK_DATABASE_USER=testlink \
  -e TESTLINK_DATABASE_PASSWORD=testlink123 \
  bitnami/testlink:latest
```

**核心概念：**
```
TestLink 层级结构：
Test Project (测试项目)
├── Test Plan (测试计划)
│   ├── Build (构建版本)
│   ├── Test Case (测试用例)
│   │   ├── Summary (摘要)
│   │   ├── Steps (步骤)
│   │   ├── Expected Results (期望结果)
│   │   └── Keywords (关键词)
│   └── Test Suite (测试套件 - 树形组织)
└── Test Report (测试报告)
```

**XML-RPC API 示例：**
```python
import xmlrpc.client

server = xmlrpc.client.ServerProxy(
    "http://testlink.example.com/lib/api/xmlrpc/v1/xmlrpc.php"
)
api_key = "your_api_key"

# 创建测试用例
def create_test_case(test_project_id, test_suite_id, title, summary, steps):
    case = server.tl.createTestCase(
        {"devKey": api_key},
        test_project_id,
        test_suite_id,
        title,
        summary,
        steps
    )
    return case

# 报告测试结果
def report_result(test_plan_id, test_case_id, build_id, status, notes=""):
    result = server.tl.reportTCResult(
        {"devKey": api_key},
        test_plan_id,
        test_case_id,
        build_id,
        {"status": status, "notes": notes}
    )
    return result
# status: 'p' (pass), 'f' (fail), 'b' (blocked)
```

### 5.2 TestRail

TestRail 是现代商业测试管理工具，界面简洁、功能强大。

**特点：**
- 直观的用例管理界面
- 灵活的测试计划与测试套件
- 测试执行进度仪表盘
- 丰富的报表与统计
- 完善的REST API
- Jira深度集成

**TestRail API：**
```python
import requests

BASE_URL = "https://your-org.testrail.io/index.php?/api/v2"
USER = "your@email.com"
PASSWORD = "your_password"

headers = {"Content-Type": "application/json"}

def add_result(test_run_id, case_id, status_id, comment=""):
    """
    status_id:
    1 - Passed
    2 - Blocked
    3 - Retest
    4 - Failed
    5 - Not Applicable
    """
    url = f"{BASE_URL}/add_result_for_case/{test_run_id}/{case_id}"
    data = {
        "status_id": status_id,
        "comment": comment
    }
    resp = requests.post(url, json=data, headers=headers,
                         auth=(USER, PASSWORD))
    return resp.json()

def add_run(project_id, suite_id, name, case_ids):
    url = f"{BASE_URL}/add_run/{project_id}"
    data = {
        "suite_id": suite_id,
        "name": name,
        "include_all": False,
        "case_ids": case_ids
    }
    resp = requests.post(url, json=data, headers=headers,
                         auth=(USER, PASSWORD))
    return resp.json()
```

## Confluence/语雀/飞书 文档协作

### 6.1 Confluence

Confluence 是 Atlassian 的企业级知识管理和协作平台，与 Jira 深度集成。

**核心功能：**
- 空间 (Space): 团队/项目专属文档空间
- 页面树: 层级化文档组织
- 模板: 丰富的页面模板（会议记录、产品需求、RFC等）
- 宏 (Macro): 功能扩展（Jira Issues、图表、代码块）
- 版本历史: 完整的修改记录
- 权限控制: 空间级/页面级权限

```
测试知识库结构：
```

### 实践4: 测试用例与需求追溯


## 总结

项目管理工具的选择建议：

| 团队类型 | 推荐方案 |
|---------|---------|
| 初创小团队 (<10人) | 禅道(免费) 或 飞书 + 多维表格 |
| 中小团队 (10-30人) | TAPD(腾讯生态) 或 禅道 |
| 中大型企业 (30+人) | Jira + Confluence (国际化) / TAPD (国内) |
| 外企/国际化团队 | Jira + Confluence + TestRail |
| 追求性价比 | 禅道(项目管理+测试一体化) |
| 重度敏捷 | Jira Software + Xray |

**核心原则：**
1. 工具要服务于流程，而不是反过来
2. 先规范化流程，再选择匹配的工具
3. 工具配置要简洁，避免过度定制
4. 重视API能力，便于与自动化测试集成
5. 定期审视工具使用效率，勇于更换不合适的工具
