## 1. 测试环境搭建标准化流程

### 通俗释义

搭建测试环境就像搬家布置新房子——你需要先检查房屋结构（操作系统），然后接通水电煤（基础运行时环境），接着摆放家具（数据库、中间件），最后验证所有设施可用（环境验证）。整个过程中要记录每一步操作，方便后续复现或排查问题。

### 详细解释

#### 环境搭建的生命周期

| 阶段 | 活动 | 输入 | 输出 | 检查点 |
|------|------|------|------|--------|
| **需求分析** | 明确测试对象所需的技术栈 | 项目技术文档、架构图 | 环境配置清单 | 技术栈版本是否与生产一致 |
| **资源申请** | 申请服务器/虚拟机/容器资源 | 环境配置清单 | 资源分配单 | CPU、内存、磁盘是否满足最低要求 |
| **系统安装** | 安装操作系统和基础软件 | 资源IP、账号 | 可登录的系统 | SSH/RDP连通性、yum/apt源 |
| **环境部署** | 按清单安装各组件 | 安装包、配置模板 | 运行中的环境 | 组件版本、端口监听状态 |
| **配置调优** | 调整参数适配测试场景 | 默认配置 | 优化后的配置 | 连接池、内存、超时等参数 |
| **验证检查** | 执行冒烟测试验证环境 | 冒烟用例 | 验证报告 | 所有核心接口/服务可用 |
| **数据准备** | 导入测试基础数据 | 数据脚本、备份文件 | 可用数据集 | 数据完整性、脱敏状态 |
| **基线备份** | 对环境做快照/备份 | 干净环境 | 环境基线 | 可快速恢复到初始状态 |

#### 环境搭建六大原则

| 原则 | 说明 | 违反后果 |
|------|------|----------|
| **一致性** | 测试环境与生产环境尽量保持一致 | 环境差异导致Bug漏测，上线后发现新问题 |
| **隔离性** | 不同测试环境之间网络隔离、数据隔离 | 环境间相互干扰，测试结果不可信 |
| **可复现性** | 任何环境问题都能通过文档步骤复现 | 环境问题定位成本极高 |
| **版本管理** | 所有组件版本记录清晰、变更可追溯 | 版本不一致导致"在我机器上能跑"的问题 |
| **最小权限** | 测试环境仅授予必要权限 | 误操作导致环境损坏 |
| **成本可控** | 根据实际需求选择资源规格 | 资源浪费或资源不足影响测试进度 |

#### 环境搭建标准化SOP（标准操作流程）

```
步骤1: 获取环境配置清单
       ├── 编程语言及版本 (e.g., Java 17, Python 3.11, Node.js 20)
       ├── 构建工具及版本 (e.g., Maven 3.9, Gradle 8.x)
       ├── 数据库及版本 (e.g., MySQL 8.0, PostgreSQL 16)
       ├── 缓存中间件及版本 (e.g., Redis 7.x)
       ├── 消息队列及版本 (e.g., Kafka 3.x, RabbitMQ 3.x)
       ├── Web服务器及版本 (e.g., Nginx 1.24, Tomcat 10)
       ├── 容器运行时及版本 (e.g., Docker 24.x)
       └── 其他依赖 (e.g., Elasticsearch, MinIO, Zookeeper)

步骤2: 检查操作系统兼容性
       ├── 确认OS版本与各组件的兼容性矩阵
       ├── 确认内核版本满足Docker等容器要求
       └── 确认系统架构 (x86_64 vs ARM64)

步骤3: 按依赖顺序安装
       ├── 先安装语言运行时（Java/Python/Node.js）
       ├── 再安装构建工具（Maven/Gradle/npm）
       ├── 再安装数据库（MySQL/PostgreSQL）
       ├── 最后安装中间件（Redis/Kafka/ES）
       └── 注：Docker环境可简化此流程

步骤4: 逐组件验证
       ├── 版本验证：java -version, python --version
       ├── 端口验证：netstat -an | grep <port>
       ├── 连接验证：mysql -u root -p, redis-cli ping
       └── 功能验证：执行CRUD操作、发送测试消息

步骤5: 编写环境文档
       ├── 环境架构拓扑图
       ├── 各组件访问地址和凭据
       ├── 常见问题排查指南
       └── 环境重置/恢复操作手册
```

## 2. Windows平台测试环境搭建

### 通俗释义

Windows是最常见的桌面操作系统，多数公司给测试工程师配发Windows电脑。在Windows上搭建测试环境需要注意路径分隔符（反斜杠`\`）、权限问题（管理员运行）、以及不少Linux原生工具的兼容性问题。

### 详细解释

#### Windows环境搭建核心要点

| 要点 | 说明 | 最佳实践 |
|------|------|----------|
| **管理员权限** | 许多安装程序需要管理员权限 | 右键"以管理员身份运行"，或将安装目录放在非系统盘 |
| **路径规范** | Windows使用`\`，部分工具支持`/` | 环境变量中统一使用`\`，脚本中优先使用`/`或`\\` |
| **换行符差异** | Windows CRLF vs Unix LF | Git配置 `git config --global core.autocrlf true` |
| **长路径限制** | 默认路径长度限制260字符 | Windows 10 1607+可在组策略中启用长路径 |
| **杀毒软件** | 可能拦截端口监听、修改hosts | 将开发目录加入白名单 |
| **包管理器** | Windows生态缺少原生包管理器 | 使用 Chocolatey 或 Scoop 管理软件包 |

#### Windows下推荐的目录结构

```
D:\DevEnvironment\
├── Java\
│   ├── jdk-17\          # JDK 17
│   └── jdk-21\          # JDK 21 (多版本共存)
├── Python\
│   ├── Python311\       # Python 3.11
│   └── Python312\       # Python 3.12
├── Maven\
│   └── apache-maven-3.9.6\
├── Gradle\
│   └── gradle-8.7\
├── NodeJS\
│   └── node-v20.12.0\
├── Database\
│   ├── MySQL\
│   └── PostgreSQL\
├── Tools\
│   ├── Git\
│   ├── DBeaver\         # 数据库管理工具
│   ├── Postman\         # 接口测试工具
│   └── RedisInsight\    # Redis可视化管理
└── Workspace\
    ├── project-a\       # 项目A工作目录
    └── project-b\       # 项目B工作目录
```

#### Chocolatey包管理器使用

Chocolatey是Windows上的包管理器，类似于Linux的apt/yum，可以一键安装和管理软件。

```powershell
# 安装Chocolatey（以管理员身份运行PowerShell）
Set-ExecutionPolicy Bypass -Scope Process -Force
[System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072
iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

# 常用安装命令
choco install jdk8 -y              # 安装JDK 8
choco install openjdk -y           # 安装最新OpenJDK
choco install python -y            # 安装Python 3
choco install nodejs-lts -y        # 安装Node.js LTS
choco install git -y               # 安装Git
choco install mysql -y             # 安装MySQL
choco install postgresql -y        # 安装PostgreSQL
choco install redis-desktop-manager -y
choco install dbeaver -y           # 数据库管理工具
choco install postman -y           # API测试工具
choco install docker-desktop -y    # Docker Desktop

# 查看已安装包
choco list --local-only
```

#### Windows特有踩坑点及解决方案

| 踩坑点 | 现象 | 解决方案 |
|--------|------|----------|
| **端口被占用** | 启动服务时报 `Address already in use` | `netstat -ano \| findstr :端口号` 查看占用进程，`taskkill /PID 进程号 /F` 杀掉 |
| **路径含空格** | `Program Files`目录下有空格导致脚本执行异常 | 安装到无空格路径，或用双引号包裹路径 |
| **环境变量不生效** | 配置了环境变量但命令行不识别 | 重启终端或执行 `refreshenv`（Chocolatey提供） |
| **DLL缺失** | 运行某些工具报缺少vcruntime140.dll等 | 安装VC++运行库：`choco install vcredist-all` |
| **hosts修改被拦截** | 杀毒软件阻止修改hosts文件 | 临时关闭杀毒软件，或用管理员身份编辑 |
| **Powershell执行策略** | 无法执行.ps1脚本 | `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` |

## 3. Mac平台测试环境搭建

### 通俗释义

Mac基于Unix，天然适合开发测试。搭配Homebrew包管理器，绝大多数测试工具都可以通过命令行一键安装。Mac与Linux服务器环境最为接近，适合需要本地模拟服务器环境的测试场景。

### 详细解释

#### Homebrew——Mac必备包管理器

Homebrew是Mac上最流行的包管理器，几乎所有开发测试工具都可以通过它安装。

```bash
# 安装Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 常用brew命令
brew search <keyword>       # 搜索软件包
brew install <package>      # 安装软件包
brew uninstall <package>    # 卸载软件包
brew upgrade <package>      # 升级软件包
brew list                   # 列出已安装的包
brew info <package>         # 查看软件包信息
brew services list          # 查看通过brew管理的后台服务
brew services start/stop/restart <service>  # 管理服务

# 测试工程师常用brew安装清单
brew install --cask temurin@17     # JDK 17 (Adoptium)
brew install --cask temurin@21     # JDK 21
brew install python@3.12           # Python 3.12
brew install node@20               # Node.js 20
brew install maven                 # Maven
brew install gradle                # Gradle
brew install mysql                 # MySQL
brew install postgresql@16         # PostgreSQL 16
brew install redis                 # Redis
brew install kafka                 # Kafka
brew install docker                # Docker
brew install --cask docker         # Docker Desktop
brew install git                   # Git
brew install jmeter                # JMeter
brew install nginx                 # Nginx

# 启动数据库和中间件服务
brew services start mysql
brew services start postgresql@16
brew services start redis
```

#### Mac环境目录建议

```
~/Develop/
├── jdk/                   # JDK通过brew管理，自动在/opt/homebrew/opt/或/Library/Java/
├── python-envs/           # Python虚拟环境集中管理
│   ├── project-a-venv/
│   └── project-b-venv/
├── node-projects/         # Node.js项目
├── maven-repo/            # Maven本地仓库 (~/.m2/repository)
└── workspaces/            # IDE工作空间
    ├── idea-projects/
    └── vscode-projects/
```

#### Mac特有优势与注意事项

| 优势 | 说明 |
|------|------|
| **Unix内核** | 与Linux服务器环境高度一致，Shell脚本直接兼容 |
| **Homebrew生态** | 几乎所有开源工具都有Homebrew配方，版本管理方便 |
| **终端体验** | iTerm2 + zsh + Oh My Zsh，开发体验流畅 |
| **Docker支持** | Docker Desktop原生支持，与Linux内核兼容性好 |
| **安全性** | SIP(系统完整性保护)机制防止系统文件被篡改 |

| 注意事项 | 说明 |
|----------|------|
| **Apple Silicon (M1/M2/M3)** | ARM架构，部分旧软件需要Rosetta 2转译；注意选择arm64版本镜像 |
| **/usr/local vs /opt/homebrew** | Intel Mac用/usr/local，Apple Silicon用/opt/homebrew |
| **Xcode Command Line Tools** | 许多工具编译依赖，必须安装：`xcode-select --install` |
| **JDK版本切换** | 使用`jenv`管理多JDK版本 |
| **端口低于1024** | macOS限制非root用户绑定1024以下端口 |

## 4. Linux平台测试环境搭建

### 通俗释义

Linux是服务器端的主流操作系统，测试工程师通常需要在Linux上搭建CI/CD环境、性能测试环境、持续集成Agent等。最常见的发行版是CentOS/RHEL系列和Ubuntu/Debian系列。

### 详细解释

#### CentOS/RHEL环境搭建（yum/dnf）

```bash
# 基础工具安装
sudo yum update -y
sudo yum install -y wget curl vim git net-tools telnet

# 安装Java
sudo yum install -y java-17-openjdk java-17-openjdk-devel
# 或手动安装特定版本
wget https://download.oracle.com/java/17/latest/jdk-17_linux-x64_bin.tar.gz
sudo tar -xzf jdk-17_linux-x64_bin.tar.gz -C /usr/local/

# 安装Python 3
sudo yum install -y python3 python3-pip

# 安装Node.js (通过NodeSource仓库)
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs

# 安装Docker
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo systemctl start docker
sudo systemctl enable docker

# 安装MySQL
sudo yum install -y https://dev.mysql.com/get/mysql80-community-release-el7-3.noarch.rpm
sudo yum install -y mysql-community-server
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 安装Redis
sudo yum install -y epel-release
sudo yum install -y redis
sudo systemctl start redis
sudo systemctl enable redis

# 安装Nginx
sudo yum install -y nginx
sudo systemctl start nginx
```

#### Ubuntu/Debian环境搭建（apt）

```bash
# 基础工具安装
sudo apt update && sudo apt upgrade -y
sudo apt install -y wget curl vim git net-tools build-essential

# 安装Java
sudo apt install -y openjdk-17-jdk

# 安装Python 3
sudo apt install -y python3 python3-pip python3-venv

# 安装Node.js (通过NodeSource)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 安装Docker
sudo apt install -y docker.io docker-compose-v2
sudo systemctl start docker
sudo systemctl enable docker
sudo usermod -aG docker $USER    # 免sudo使用docker

# 安装MySQL
sudo apt install -y mysql-server
sudo systemctl start mysql

# 安装Redis
sudo apt install -y redis-server
sudo systemctl start redis-server

# 安装Nginx
sudo apt install -y nginx
sudo systemctl start nginx
```

#### Linux环境目录规范

```
/opt/                           # 手动安装的软件目录
├── jdk-17/
├── jdk-21/
├── maven-3.9.6/
├── gradle-8.7/
└── apache-jmeter-5.6/

/usr/local/                     # 本地编译安装的软件
├── bin/
├── lib/
└── etc/

/data/                          # 数据目录
├── mysql/
├── redis/
├── logs/                       # 日志集中存放
└── backups/                    # 备份目录

/home/<user>/                   # 用户目录
├── .m2/repository/             # Maven本地仓库
├── .npm/                       # npm缓存
└── projects/                   # 项目工作目录
```

#### Linux环境安全配置要点

```bash
# 1. 关闭防火墙或开放特定端口
sudo firewall-cmd --add-port=8080/tcp --permanent   # CentOS
sudo ufw allow 8080/tcp                               # Ubuntu

# 2. 配置SELinux（CentOS）- 测试环境可临时关闭
sudo setenforce 0
sudo sed -i 's/SELINUX=enforcing/SELINUX=disabled/g' /etc/selinux/config

# 3. 配置limits（文件句柄、进程数）
sudo vi /etc/security/limits.conf
# 添加：
# * soft nofile 65536
# * hard nofile 65536
# * soft nproc 32768
# * hard nproc 32768

# 4. 配置sysctl（网络参数调优）
sudo vi /etc/sysctl.conf
# 添加：
# net.core.somaxconn = 1024
# vm.swappiness = 1
# vm.overcommit_memory = 1
sudo sysctl -p
```

## 5. Java环境搭建详细指南

### 通俗释义

Java环境是测试工程师最常搭建的环境之一，因为大量企业级项目使用Java技术栈。JDK是Java开发工具包，Maven/Gradle是项目构建工具，IDEA是主流IDE。三者配合构成完整的Java开发和测试环境。

### 详细解释

#### JDK安装与配置

**JDK版本选择建议：**

| 版本 | LTS | 推荐场景 |
|------|-----|----------|
| JDK 8 | 是 | 遗留老项目、大数据生态（Hadoop/Spark） |
| JDK 11 | 是 | 部分中间件项目，从JDK8迁移的过渡版本 |
| JDK 17 | 是 | 当前主流，绝大多数新项目推荐 |
| JDK 21 | 是 | 最新LTS，适合追求新特性的项目 |

**Windows安装步骤：**
```
1. 下载JDK
   推荐Adoptium (Eclipse Temurin): https://adoptium.net/
   或Oracle JDK: https://www.oracle.com/java/technologies/downloads/
   或Azul Zulu: https://www.azul.com/downloads/

2. 运行安装程序
   - 安装路径建议：D:\DevEnvironment\Java\jdk-17
   - 勾选"设置JAVA_HOME变量"（如有此选项）

3. 手动配置环境变量
   系统变量新增：
   JAVA_HOME = D:\DevEnvironment\Java\jdk-17
   Path 新增：%JAVA_HOME%\bin
   （建议将%JAVA_HOME%\bin放在Path最前面，避免其他程序自带Java干扰）

   CLASSPATH = .;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar
   （JDK 9+可省略CLASSPATH配置）

4. 验证安装
   java -version
   javac -version
   echo %JAVA_HOME%
```

**Mac/Linux安装步骤：**
```bash
# Mac (Homebrew)
brew install --cask temurin@17

# Linux (手动安装，支持多版本共存)
sudo tar -xzf jdk-17.0.10_linux-x64_bin.tar.gz -C /usr/local/
sudo ln -s /usr/local/jdk-17.0.10 /usr/local/jdk-17

# 配置环境变量（~/.bashrc 或 ~/.zshrc）
export JAVA_HOME=/usr/local/jdk-17
export PATH=$JAVA_HOME/bin:$PATH
export CLASSPATH=.:$JAVA_HOME/lib

# 多JDK版本切换工具 jenv
git clone https://github.com/jenv/jenv.git ~/.jenv
echo 'export PATH="$HOME/.jenv/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(jenv init -)"' >> ~/.bashrc
source ~/.bashrc
jenv add /usr/local/jdk-17
jenv add /usr/local/jdk-21
jenv global 17       # 设置默认JDK版本
jenv local 21        # 为当前目录设置特定JDK版本
```

#### Maven配置

**Mac：**

#### Redis核心配置（redis.conf）

```conf
# 网络配置
bind 127.0.0.1              # 测试环境仅本地访问
port 6379
tcp-backlog 511
timeout 300

# 持久化配置（测试环境通常关闭或使用RDB即可）
save 900 1                  # 900秒内有1次修改就保存
save 300 10                 # 300秒内有10次修改就保存
save 60 10000               # 60秒内有10000次修改就保存
# 测试环境可以注释掉save行，不做持久化

# 内存管理
maxmemory 256mb             # 限制最大内存
maxmemory-policy allkeys-lru  # 内存满后的淘汰策略

# 日志
loglevel notice
logfile "/var/log/redis/redis.log"

# 慢查询日志
slowlog-log-slower-than 10000  # 超过10毫秒记录
slowlog-max-len 128

# 连接限制
maxclients 100

# 密码认证（测试环境可选）
requirepass TestRedis123
```

```
   默认账号：guest / guest
```

```bash
sudo rabbitmqctl add_user tester Test123456
sudo rabbitmqctl set_user_tags tester administrator
sudo rabbitmqctl set_permissions -p / tester ".*" ".*" ".*"
```

#### Kafka安装与配置

**前提：Kafka依赖ZooKeeper（Kafka 3.3+可使用KRaft模式替代）**

```bash
# 1. 下载Kafka
wget https://dlcdn.apache.org/kafka/3.6.0/kafka_2.13-3.6.0.tgz
tar -xzf kafka_2.13-3.6.0.tgz -C /usr/local/

# 2. 启动ZooKeeper（使用Kafka自带的）
cd /usr/local/kafka_2.13-3.6.0
bin/zookeeper-server-start.sh config/zookeeper.properties &

# 3. 启动Kafka
bin/kafka-server-start.sh config/server.properties &

# 4. 创建测试Topic
bin/kafka-topics.sh --create --topic test-topic --bootstrap-server localhost:9092 --partitions 3 --replication-factor 1

# 5. 测试发送消息
bin/kafka-console-producer.sh --topic test-topic --bootstrap-server localhost:9092

# 6. 测试消费消息（新开终端）
bin/kafka-console-consumer.sh --topic test-topic --from-beginning --bootstrap-server localhost:9092
```

**Kafka核心配置（config/server.properties）：**
```properties
# Broker标识
broker.id=0

# 监听地址
listeners=PLAINTEXT://localhost:9092
advertised.listeners=PLAINTEXT://localhost:9092

# 日志目录
log.dirs=/tmp/kafka-logs

# Topic默认分区数
num.partitions=3

# 日志保留策略（测试环境可缩短）
log.retention.hours=168

# ZooKeeper连接
zookeeper.connect=localhost:2181
```

**RabbitMQ vs Kafka 对比：**

| 对比维度 | RabbitMQ | Kafka |
|----------|----------|-------|
| **定位** | 消息队列 | 分布式流平台 |
| **协议** | AMQP 0-9-1 | 自定义协议 |
| **吞吐量** | 万级/秒 | 百万级/秒 |
| **消息持久化** | 支持 | 天然持久化 |
| **消费模式** | Push | Pull |
| **适用场景** | 业务异步解耦、任务分发 | 日志收集、流处理、事件溯源 |
| **路由能力** | 灵活（Exchange/Queue/Binding） | 基于Topic分区 |
| **测试难度** | 较低 | 较高 |
| **管理界面** | 内置（15672端口） | 需额外安装（如Kafka UI） |

## 11. Docker环境搭建

### 通俗释义

Docker是测试环境搭建的"终极解决方案"——通过容器化技术，将环境配置固化为代码（Dockerfile/docker-compose.yml），实现"一次构建，处处运行"。对于测试工程师来说，Docker可以极大简化环境搭建流程，消除"环境不一致"导致的假Bug。

### 详细解释

#### Docker Desktop安装

**Windows：**
```

   推荐使用WSL2后端（性能更好）

   https://www.docker.com/products/docker-desktop/

   - Settings -> Docker Engine -> 配置镜像加速

4. 镜像加速配置（Docker Engine配置）：
   {
     "registry-mirrors": [
       "https://docker.m.daocloud.io",
       "https://dockerhub.timeweb.cloud"
     ]
   }

5. 验证安装：
   docker --version
   docker run hello-world
```

**Mac：**
```bash
brew install --cask docker
# 启动Docker Desktop应用
# 在Settings中配置镜像加速（同Windows）
```

**Linux：**

## 12. 环境变量配置规范

### 通俗释义

环境变量是操作系统中用来存储配置信息的全局变量。在测试环境中，环境变量用于指定软件安装路径（如`JAVA_HOME`）、服务连接信息（如`DATABASE_URL`）、以及各种开关配置。规范的环境变量管理是环境可维护性的基础。

### 详细解释

#### 三平台环境变量配置方法

**Windows环境变量：**
```
配置路径：控制面板 -> 系统 -> 高级系统设置 -> 环境变量
或：设置 -> 系统 -> 关于 -> 高级系统设置 -> 环境变量

分为两类：
- 用户变量：仅当前用户生效
- 系统变量：所有用户生效（需管理员权限）

常见配置：
JAVA_HOME    = D:\DevEnvironment\Java\jdk-17
MAVEN_HOME   = D:\DevEnvironment\Maven\apache-maven-3.9.6
GRADLE_HOME  = D:\DevEnvironment\Gradle\gradle-8.7
NODE_PATH    = D:\DevEnvironment\npm-global\node_modules
Path 追加：
  %JAVA_HOME%\bin
  %MAVEN_HOME%\bin
  %GRADLE_HOME%\bin
  D:\DevEnvironment\Python\Python311
  D:\DevEnvironment\Python\Python311\Scripts
  D:\DevEnvironment\NodeJS\
  D:\DevEnvironment\npm-global
```

**Mac环境变量（~/.zshrc 或 ~/.bash_profile）：**
```bash
# Java
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
export PATH=$JAVA_HOME/bin:$PATH

# Maven
export MAVEN_HOME=/usr/local/maven
export PATH=$MAVEN_HOME/bin:$PATH

# Gradle
export GRADLE_HOME=/usr/local/gradle
export PATH=$GRADLE_HOME/bin:$PATH

# Python
export PATH="/usr/local/opt/python@3.11/bin:$PATH"

# Node.js
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"

# Docker
export DOCKER_HOST=unix:///var/run/docker.sock

# 全局
export LANG=en_US.UTF-8
export LC_ALL=en_US.UTF-8
```

**Linux环境变量（/etc/profile 或 ~/.bashrc）：**
```bash
# 系统级：/etc/profile.d/test-env.sh
export JAVA_HOME=/usr/local/jdk-17
export MAVEN_HOME=/usr/local/maven
export GRADLE_HOME=/usr/local/gradle
export PATH=$JAVA_HOME/bin:$MAVEN_HOME/bin:$GRADLE_HOME/bin:$PATH
```

#### 测试环境专用环境变量规范

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `JAVA_HOME` | JDK安装路径 | `D:\DevEnvironment\Java\jdk-17` |
| `MAVEN_HOME` | Maven安装路径 | `D:\DevEnvironment\Maven\apache-maven-3.9.6` |
| `GRADLE_HOME` | Gradle安装路径 | `D:\DevEnvironment\Gradle\gradle-8.7` |
| `PYTHON_HOME` | Python安装路径 | `D:\DevEnvironment\Python\Python311` |
| `NODE_HOME` | Node.js安装路径 | `D:\DevEnvironment\NodeJS` |
| `DB_HOST` | 测试数据库地址 | `localhost` 或 `192.168.1.100` |
| `DB_PORT` | 测试数据库端口 | `3306`（MySQL）或 `5432`（PostgreSQL） |
| `DB_USER` | 测试数据库用户 | `tester` |
| `DB_PASSWORD` | 测试数据库密码 | 注意：生产环境勿用明文，使用密钥管理 |
| `REDIS_HOST` | Redis地址 | `localhost` |
| `REDIS_PORT` | Redis端口 | `6379` |
| `MQ_HOST` | 消息队列地址 | `localhost` |
| `TEST_ENV` | 测试环境标识 | `local` / `dev` / `staging` |
| `TEST_BASE_URL` | 被测应用基础URL | `http://localhost:8080` |

#### 环境变量管理最佳实践

1. **使用.env文件管理项目级环境变量**（不提交到Git）
2. **敏感信息（密码/Token/密钥）不要写在环境变量中**，使用密钥管理服务
3. **环境变量命名使用大写+下划线**，如`DB_HOST`而非`dbHost`
4. **在CI/CD中使用Secret变量**管理敏感环境变量
5. **为每个环境维护独立的配置文件**，如`application-dev.yml`、`application-test.yml`
