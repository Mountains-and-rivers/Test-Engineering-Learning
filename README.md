## 🎯 使用指南

### 学习路线建议

| 阶段 | 模块 | 目标 |
|------|------|------|
| **入门** | 01 → 02 → 12 | 掌握测试基础理论、四层测试体系、搭建测试环境 |
| **进阶** | 00 → 03 → 10 | 精通黑盒/白盒/灰盒方法、各专项测试、熟练使用工具栈 |
| **高级** | 04 → 05 → 06 | 搭建自动化工程体系、掌握缺陷管理和应急处理、编写测试策略 |
| **专家** | 07 → 09 → 11 | 建设质量度量KPI体系、复盘实战项目、管理测试团队 |
| **面试/晋升** | 08 → 13 | 突击面试题库、打磨项目话术、规划职业进阶路径 |

### 推荐学习方式

1. **按目录顺序循序渐进**：每个目录按编号顺序阅读，由浅入深
2. **结合实战项目**：阅读 09_实战案例库 中的项目，边学边练
3. **善用 FAQ**：每个模块的 FAQ 文件（如常见问题/问题汇总等）收录了常见问题，遇到障碍优先查找
4. **面试前突击**：08_面试题库 + 06_项目话术 = 完整的面试准备体系

### 文档特色

- 📘 **系统化**：14大模块覆盖测试工程全维度，无死角
- 📝 **实战化**：大部分知识点配备真实场景案例（FAQ/问题汇总类文件侧重诊断思路，主知识文件标配5+案例）
- 📊 **结构化**：丰富的对比表格、流程图、checklist，便于检索
- 🔧 **工具化**：50+主流工具的详细配置和使用教程
- 💼 **企业级**：基于大厂测试工程实践，可直接落地复用
