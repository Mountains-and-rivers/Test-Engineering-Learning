# 框架配置层

> **框架配置层**（Framework Configuration Layer）：自动化测试框架的基石层，负责管理测试框架的所有配置信息，包括环境配置、参数管理、配置中心集成和敏感信息安全存储。配置层的设计质量直接影响测试框架的可维护性、可移植性和安全性。

## 1. 自动化测试框架架构设计

### 通俗释义

自动化测试框架就像盖房子的地基和脚手架——决定了上层代码怎么写、工具怎么用、测试怎么跑。一个好的框架让测试人员只需要关注"测什么"，而不是"怎么测"。

### 详细解释

#### 框架分层架构总览

```
┌─────────────────────────────────────────────────────────────┐
│                    测试用例层 (Test Case Layer)               │
│  用例组织 | 数据驱动 | 标签分组 | 失败重试 | 并发执行          │
├─────────────────────────────────────────────────────────────┤
│                    业务封装层 (Business Layer)                │
│  Page Object | API Object | 业务流程封装 | 公共断言           │
├─────────────────────────────────────────────────────────────┤
│                    夹具层 (Fixture Layer)                     │
│  测试数据准备 | 数据工厂 | 数据清理 | 数据库快照               │
├─────────────────────────────────────────────────────────────┤
│                    工具封装层 (Tool Layer)                     │
│  HTTP客户端 | 数据库操作 | 文件操作 | 日志 | 加密 | 通知       │
├─────────────────────────────────────────────────────────────┤
│                    框架配置层 (Config Layer)     ← 本章重点    │
│  环境配置 | 参数管理 | 配置中心 | 敏感信息 | 框架开关          │
├─────────────────────────────────────────────────────────────┤
│                    驱动层 (Driver Layer)                      │
│  WebDriver | Appium | HTTP Client | gRPC | WebSocket         │
├─────────────────────────────────────────────────────────────┤
│                    基础设施层 (Infrastructure Layer)           │
│  Jenkins | Docker | K8s | Selenium Grid | 报告服务            │
└─────────────────────────────────────────────────────────────┘
```

#### 框架设计核心原则

| 原则 | 说明 | 反面示例 |
|------|------|----------|
| **配置与代码分离** | 配置信息不硬编码在代码中，通过外部文件/环境变量加载 | `String url = "https://test.example.com"` 写死在类中 |
| **约定优于配置** | 提供合理默认值，减少必填配置项 | 每个测试类都要显式声明所有依赖 |
| **单一数据源** | 每项配置只在一个地方定义，避免多处重复 | 同一个数据库地址在3个配置文件中各写一遍 |
| **环境可移植** | 切换环境只需改一处配置（环境变量或启动参数） | 切环境需要改10个文件 |
| **安全第一** | 敏感信息加密存储，日志中脱敏输出 | 密码明文写在配置文件里提交到Git |

#### 框架配置的常见技术选型

| 配置方式 | 适用场景 | 优点 | 缺点 |
|----------|----------|------|------|
| **YAML文件** | 中大型项目，配置结构复杂 | 层次清晰、支持注释、可读性强 | 格式敏感（缩进）、解析库依赖 |
| **Properties文件** | Java项目，简单键值对配置 | Java原生支持、简单直观 | 不支持层级结构、不支持列表 |
| **JSON文件** | 前端/Node.js项目 | 通用性强、工具链丰富 | 不支持注释、括号嵌套深 |
| **环境变量** | 容器化部署、敏感信息 | 最安全、12-Factor标准 | 不直观、调试不便 |
| **配置中心** | 微服务、多项目共享配置 | 动态更新、版本管理、灰度发布 | 引入额外依赖、网络依赖 |

## 2. 配置文件管理

### 通俗释义

配置文件是测试框架的"遥控器"——通过修改配置文件就能控制测试框架的行为，而不需要改代码。YAML、JSON、Properties是最主流的三种配置格式。

### 详细解释

#### YAML配置详解

```yaml
# config/test-config.yaml
framework:
  name: "EnterpriseTestFramework"
  version: "2.3.1"
  
environment:
  active: "test"  # 当前激活的环境
  
# 测试执行配置
execution:
  parallel:
    enabled: true
    thread_count: 8
    per_core_thread_count: false
  retry:
    enabled: true
    max_retries: 2
    retry_on: ["AssertionError", "TimeoutError", "NetworkError"]
  timeout:
    test_case: 300      # 单个用例超时(秒)
    suite: 1800          # 套件超时(秒)
    element_wait: 10     # 元素等待超时(秒)

# 浏览器配置
browser:
  type: "chrome"         # chrome/firefox/edge/safari
  headless: false
  window_size: "1920x1080"
  options:
    - "--disable-gpu"
    - "--no-sandbox"
    - "--disable-dev-shm-usage"

# 数据库配置(按环境)
databases:
  test:
    mysql:
      host: "10.0.1.100"
      port: 3306
      database: "test_db"
      pool:
        min_size: 5
        max_size: 20
    redis:
      host: "10.0.1.101"
      port: 6379
      db: 0

# 日志配置
logging:
  level: "DEBUG"         # TRACE/DEBUG/INFO/WARN/ERROR
  console: true
  file:
    enabled: true
    path: "./logs/"
    max_size: "50MB"
    backup_count: 10
```

#### Java中读取YAML配置

```java
// 使用SnakeYAML读取配置
public class YamlConfigLoader {
    private static Map<String, Object> config;
    
    static {
        Yaml yaml = new Yaml();
        try (InputStream in = new FileInputStream("config/test-config.yaml")) {
            config = yaml.load(in);
        } catch (IOException e) {
            throw new RuntimeException("Failed to load config", e);
        }
    }
    
    @SuppressWarnings("unchecked")
    public static <T> T get(String path) {
        String[] keys = path.split("\\.");
        Map<String, Object> current = config;
        for (int i = 0; i < keys.length - 1; i++) {
            current = (Map<String, Object>) current.get(keys[i]);
        }
        return (T) current.get(keys[keys.length - 1]);
    }
    
    // 使用示例
    public static void main(String[] args) {
        String env = YamlConfigLoader.get("environment.active");
        int timeout = YamlConfigLoader.get("execution.timeout.test_case");
        System.out.println("当前环境: " + env + ", 用例超时: " + timeout + "秒");
    }
}
```

#### 配置文件加载优先级策略

```
配置文件加载优先级（从高到低）：
1. 命令行参数          -Ddb.host=192.168.1.1
2. 操作系统环境变量     DB_HOST=192.168.1.1
3. 外部配置文件         config/application-{env}.yaml
4. 默认配置文件         config/application.yaml
5. 代码中硬编码默认值   private String host = "localhost"
```

**加载优先级实现代码：**

```java
public class ConfigManager {
    private static final int PRIORITY_CLI = 5;
    private static final int PRIORITY_ENV = 4;
    private static final int PRIORITY_EXTERNAL = 3;
    private static final int PRIORITY_DEFAULT = 2;
    private static final int PRIORITY_CODE = 1;
    
    public static String getProperty(String key, String defaultValue) {
        // 1. 尝试命令行参数
        String cliValue = System.getProperty(key);
        if (cliValue != null) return cliValue;
        
        // 2. 尝试环境变量
        String envValue = System.getenv(key.replace(".", "_").toUpperCase());
        if (envValue != null) return envValue;
        
        // 3. 尝试外部配置文件
        String fileValue = ExternalConfig.get(key);
        if (fileValue != null) return fileValue;
        
        // 4. 返回默认值
        return defaultValue;
    }
}
```

## 3. 多环境配置切换

### 通俗释义

多环境配置切换让你用一套代码在不同的环境（开发、测试、预发布、生产）中运行，只需改一个参数——比如 `env=test` 或 `env=prod`，框架就会自动加载对应的数据库地址、接口域名、账号密码等。

### 详细解释

#### 多环境配置目录结构

```
config/
├── application.yaml              # 主配置（公共配置 + 默认值）
├── application-dev.yaml          # 开发环境
├── application-test.yaml         # 测试环境
├── application-staging.yaml      # 预发布环境
├── application-prod.yaml         # 生产环境
├── datasource/
│   ├── dev-datasource.yaml       # 开发环境数据源
│   ├── test-datasource.yaml      # 测试环境数据源
│   └── prod-datasource.yaml      # 生产环境数据源
└── accounts/
    ├── test-accounts.yaml        # 测试账号（加密存储）
    └── prod-accounts.yaml        # 生产账号（加密存储）
```

#### 环境切换实现方案

**方案二：Python动态加载模式**

```python
# config/env_config.py
import os
import yaml
from pathlib import Path

class EnvironmentConfig:
    """多环境配置管理器"""
    
    def __init__(self, env: str = None):
        # 环境判定优先级：参数 > 环境变量 > "test"
        self.env = env or os.getenv("TEST_ENV", "test")
        self.config_dir = Path(__file__).parent
        self._config = {}
        self._load_config()
    
    def _load_config(self):
        """加载配置：先加载公共配置，再加载环境专属配置覆盖"""
        # 1. 加载公共配置
        common_file = self.config_dir / "application.yaml"
        if common_file.exists():
            with open(common_file, 'r', encoding='utf-8') as f:
                self._config = yaml.safe_load(f) or {}
        
        # 2. 加载环境专属配置（覆盖公共配置）
        env_file = self.config_dir / f"application-{self.env}.yaml"
        if env_file.exists():
            with open(env_file, 'r', encoding='utf-8') as f:
                env_config = yaml.safe_load(f) or {}
                self._deep_update(self._config, env_config)
        
        # 3. 环境变量覆盖（最高优先级）
        self._apply_env_overrides()
    
    def _deep_update(self, base: dict, override: dict):
        """深度合并配置字典"""
        for key, value in override.items():
            if isinstance(value, dict) and key in base and isinstance(base[key], dict):
                self._deep_update(base[key], value)
            else:
                base[key] = value
    
    def _apply_env_overrides(self):
        """应用环境变量覆盖"""
        # 例如：API_BASE_URL -> api.base-url 的映射
        override_map = {
            'API_BASE_URL': 'api.base-url',
            'DB_HOST': 'datasource.host',
            'DB_PORT': 'datasource.port',
            'LOG_LEVEL': 'logging.level',
        }
        for env_key, config_path in override_map.items():
            env_value = os.getenv(env_key)
            if env_value:
                self._set_nested(config_path, env_value)
    
    def get(self, path: str, default=None):
        """通过点分隔路径获取配置值"""
        keys = path.split('.')
        value = self._config
        for key in keys:
            if isinstance(value, dict):
                value = value.get(key)
            else:
                return default
            if value is None:
                return default
        return value
    
    def _set_nested(self, path: str, value):
        """通过点分隔路径设置配置值"""
        keys = path.split('.')
        target = self._config
        for key in keys[:-1]:
            if key not in target:
                target[key] = {}
            target = target[key]
        target[keys[-1]] = value


# 使用示例
config = EnvironmentConfig()
print(f"当前环境: {config.env}")
print(f"API地址: {config.get('api.base-url')}")
print(f"数据库: {config.get('datasource.host')}:{config.get('datasource.port')}")
```

## 4. 全局参数与局部参数管理

### 通俗释义

全局参数是所有测试用例共享的配置（如API地址、数据库连接），局部参数是单个用例或套件特有的配置（如特定测试数据、用例开关）。区分二者可以避免"改一个用例的配置结果影响了其他用例"的尴尬。

### 详细解释

#### 参数作用域层级

```
┌──────────────────────────────────────────────┐
│ 全局参数 (Global)                             │
│   ├── 系统级: JVM参数、环境变量               │
│   ├── 框架级: 框架配置文件                    │
│   └── 项目级: 项目配置文件                    │
├──────────────────────────────────────────────┤
│ 套件参数 (Suite)                              │
│   ├── 测试套件级配置                          │
│   └── 分组级配置                              │
├──────────────────────────────────────────────┤
│ 用例参数 (Test Case)                          │
│   ├── 用例级配置                              │
│   └── 数据驱动参数                            │
└──────────────────────────────────────────────┘
```

#### Java全局参数管理实现

```java
// 全局参数管理
@Component
public class GlobalConfig {
    
    @Value("${api.base-url}")
    private String apiBaseUrl;
    
    @Value("${execution.timeout.test_case:300}")
    private int defaultTimeout;
    
    @Value("${execution.retry.max_retries:2}")
    private int maxRetries;
    
    // 全局参数Map（支持运行时动态参数）
    private static final ConcurrentHashMap<String, Object> GLOBAL_PARAMS = new ConcurrentHashMap<>();
    
    public static void set(String key, Object value) {
        GLOBAL_PARAMS.put(key, value);
    }
    
    public static Object get(String key) {
        return GLOBAL_PARAMS.get(key);
    }
    
    @SuppressWarnings("unchecked")
    public static <T> T get(String key, Class<T> type) {
        return (T) GLOBAL_PARAMS.get(key);
    }
    
    public static void clear() {
        GLOBAL_PARAMS.clear();
    }
}
```

#### Python参数作用域管理

```python
# config/param_manager.py
import threading
from typing import Any, Dict, Optional
from contextlib import contextmanager

class ParamScope:
    """参数作用域枚举"""
    GLOBAL = "global"       # 全局共享
    SUITE = "suite"         # 套件共享
    CLASS = "class"         # 类共享
    METHOD = "method"       # 方法独立

class ParamManager:
    """分层参数管理器"""
    
    _instance = None
    
    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._init()
        return cls._instance
    
    def _init(self):
        # 全局参数（进程级别）
        self._global_params: Dict[str, Any] = {}
        # 套件参数（线程级别）
        self._suite_params: threading.local = threading.local()
    
    def set(self, key: str, value: Any, scope: ParamScope = ParamScope.GLOBAL):
        """设置参数到指定作用域"""
        if scope == ParamScope.GLOBAL:
            self._global_params[key] = value
        elif scope == ParamScope.SUITE:
            if not hasattr(self._suite_params, 'data'):
                self._suite_params.data = {}
            self._suite_params.data[key] = value
    
    def get(self, key: str, default: Any = None,
            scope_hint: Optional[ParamScope] = None) -> Any:
        """获取参数（自动从局部到全局查找）"""
        if scope_hint == ParamScope.SUITE:
            if hasattr(self._suite_params, 'data'):
                return self._suite_params.data.get(key, default)
        # 默认从全局获取
        return self._global_params.get(key, default)
    
    def get_or_default(self, key: str, local_value: Any, 
                       default: Any) -> Any:
        """局部值优先，没有则用全局，再没有用默认值"""
        if local_value is not None:
            return local_value
        return self.get(key, default)
    
    def clear_scope(self, scope: ParamScope):
        """清理指定作用域的参数"""
        if scope == ParamScope.GLOBAL:
            self._global_params.clear()
        elif scope == ParamScope.SUITE and hasattr(self._suite_params, 'data'):
            self._suite_params.data.clear()


# 使用示例
param_mgr = ParamManager()

# 设置全局参数（框架启动时）
param_mgr.set("api.base_url", "https://api-test.example.com", ParamScope.GLOBAL)
param_mgr.set("default.timeout", 30, ParamScope.GLOBAL)

# 测试套件中使用套件级参数
param_mgr.set("suite.name", "UserModule", ParamScope.SUITE)
```

## 5. 配置中心集成

### 通俗释义

配置中心是一个独立的服务，专门管理所有应用的配置。当你的自动化测试框架需要和几十个微服务共享配置时，配置中心能让所有配置"一处修改、处处生效"，还能实现配置的动态刷新和灰度发布。

### 详细解释

#### Python Nacos集成

```python
# config/nacos_client.py
import nacos
import threading
import yaml
import logging

logger = logging.getLogger(__name__)

class NacosConfigManager:
    """Nacos配置管理客户端"""
    
    def __init__(self, server_addr: str, namespace: str, 
                 data_id: str, group: str = "DEFAULT_GROUP"):
        self.server_addr = server_addr
        self.namespace = namespace
        self.data_id = data_id
        self.group = group
        self._config_data = {}
        self._lock = threading.RLock()
        
        # 初始化Nacos客户端
        self.client = nacos.NacosClient(
            server_addresses=server_addr,
            namespace=namespace
        )
        
        # 加载初始配置
        self._load_config()
        
        # 添加配置监听
        self.client.add_config_watcher(
            data_id=data_id,
            group=group,
            cb=self._on_config_change
        )
    
    def _load_config(self):
        """从Nacos加载配置"""
        try:
            raw_config = self.client.get_config(
                data_id=self.data_id,
                group=self.group
            )
            if raw_config:
                with self._lock:
                    # 根据data_id后缀判断格式
                    if self.data_id.endswith('.yaml') or self.data_id.endswith('.yml'):
                        self._config_data = yaml.safe_load(raw_config) or {}
                    else:
                        # Properties格式解析
                        self._config_data = self._parse_properties(raw_config)
                logger.info(f"Nacos配置加载成功: {self.data_id}")
        except Exception as e:
            logger.error(f"Nacos配置加载失败: {e}")
    
    def _on_config_change(self, config: dict):
        """配置变更回调"""
        logger.info(f"Nacos配置变更: {config}")
        with self._lock:
            content = config.get('content', '')
            if content:
                if self.data_id.endswith(('.yaml', '.yml')):
                    self._config_data = yaml.safe_load(content) or {}
                else:
                    self._config_data = self._parse_properties(content)
    
    def _parse_properties(self, raw: str) -> dict:
        """解析Properties格式"""
        result = {}
        for line in raw.split('\n'):
            line = line.strip()
            if line and not line.startswith('#') and '=' in line:
                key, value = line.split('=', 1)
                result[key.strip()] = value.strip()
        return result
    
    def get(self, key: str, default=None):
        """获取配置项（支持点分隔的嵌套路径）"""
        with self._lock:
            keys = key.split('.')
            value = self._config_data
            for k in keys:
                if isinstance(value, dict):
                    value = value.get(k)
                else:
                    return default
                if value is None:
                    return default
            return value

    data_id="automation-config.yaml",
    group="AUTOMATION"
)
print(f"API地址: {nacos_mgr.get('api.base-url')}")
```

## 6. 敏感信息管理

### 通俗释义

敏感信息（密码、密钥、Token、证书）绝不能明文写在配置文件里提交到Git仓库——这就像把家门钥匙放在门口地毯下面。需要加密存储、解密使用，且解密后的值不应出现在日志中。

### 详细解释

#### 敏感信息分层保护策略

```
┌─────────────────────────────────────────────────────┐
│ Level 1: 代码/配置文件中明文          ← 绝对禁止     │
│ Level 2: 配置文件中Base64编码         ← 不推荐       │
│ Level 3: 配置文件加密 + 密钥在环境变量  ← 基本安全    │
│ Level 4: 配置中心加密 + 密钥管理服务   ← 推荐         │
│ Level 5: 密钥管理服务(KMS/HashiCorp Vault) ← 企业级  │
└─────────────────────────────────────────────────────┘
```

#### Jasypt加密方案（Java）

```java
// 使用Jasypt加密敏感配置
import org.jasypt.encryption.pbe.StandardPBEStringEncryptor;
import org.jasypt.encryption.pbe.config.EnvironmentPBEConfig;

public class SensitiveConfigProtector {
    
    // 加密密钥从环境变量获取，不写在代码中
    private static final String ENC_PASSWORD = System.getenv("JASYPT_ENCRYPTOR_PASSWORD");
    
    /**
     * 加密明文
     * 使用示例：java -cp jasypt.jar ... input="realPassword" password="yourKey"
     */
    public static String encrypt(String plainText) {
        StandardPBEStringEncryptor encryptor = new StandardPBEStringEncryptor();
        encryptor.setPassword(ENC_PASSWORD);
        encryptor.setAlgorithm("PBEWithHMACSHA512AndAES_256");
        return "ENC(" + encryptor.encrypt(plainText) + ")";
    }
    
    /**
     * 解密
     */
    public static String decrypt(String encryptedText) {
        if (!encryptedText.startsWith("ENC(") || !encryptedText.endsWith(")")) {
            return encryptedText; // 非加密格式，直接返回
        }
        String actualEncrypted = encryptedText.substring(4, encryptedText.length() - 1);
        StandardPBEStringEncryptor encryptor = new StandardPBEStringEncryptor();
        encryptor.setPassword(ENC_PASSWORD);
        encryptor.setAlgorithm("PBEWithHMACSHA512AndAES_256");
        return encryptor.decrypt(actualEncrypted);
    }
}
```

#### Python加密方案

```python
# config/crypto.py
import os
import base64
import hashlib
from cryptography.fernet import Fernet
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2

class SensitiveConfigManager:
    """敏感配置加密管理器"""
    
    def __init__(self, master_key: str = None):
        """
        初始化加密管理器
        :param master_key: 主密钥，不传则从环境变量读取
        """
        self.master_key = master_key or os.getenv('CONFIG_MASTER_KEY', 'default-dev-key')
        self._fernet = self._create_fernet()
    
    def _create_fernet(self) -> Fernet:
        """创建Fernet加密器"""
        # 使用PBKDF2派生密钥
        kdf = PBKDF2(
            algorithm=hashes.SHA256(),
            length=32,
            salt=b'test-automation-salt',  # 生产环境应随机生成并安全存储
            iterations=480000,
        )
        key = base64.urlsafe_b64encode(kdf.derive(self.master_key.encode()))
        return Fernet(key)
    
    def encrypt(self, plain_text: str) -> str:
        """加密敏感信息"""
        encrypted = self._fernet.encrypt(plain_text.encode())
        return base64.urlsafe_b64encode(encrypted).decode()
    
    def decrypt(self, encrypted_text: str) -> str:
        """解密敏感信息"""
        encrypted = base64.urlsafe_b64decode(encrypted_text.encode())
        decrypted = self._fernet.decrypt(encrypted)
        return decrypted.decode()
    
    def load_encrypted_config(self, config_file: str) -> dict:
        """加载加密的配置文件"""
        import yaml
        with open(config_file, 'r', encoding='utf-8') as f:
            raw_config = yaml.safe_load(f)
        
        return self._decrypt_nested(raw_config)
    
    def _decrypt_nested(self, data: dict) -> dict:
        """递归解密嵌套配置中的ENC()标记值"""
        result = {}
        for key, value in data.items():
            if isinstance(value, dict):
                result[key] = self._decrypt_nested(value)
            elif isinstance(value, str) and value.startswith('ENC(') and value.endswith(')'):
                encrypted = value[4:-1]
                result[key] = self.decrypt(encrypted)
            else:
                result[key] = value
        return result


# 加密配置文件的生成
def generate_encrypted_config():

    secrets = {
        'datasource': {

        },
        'api': {
            'client_secret': 'ENC(' + mgr.encrypt('sk-live-abcdef123456') + ')',
        },
        'redis': {
            'password': 'ENC(' + mgr.encrypt('RedisP@ss789') + ')',
        }
    }
    
    import yaml

# 日志脱敏Filter
class SensitiveLogFilter(logging.Filter):
    """敏感信息日志过滤器"""
    
    SENSITIVE_KEYS = ['password', 'secret', 'token', 'api_key', 'private_key']
    
    def filter(self, record):
        # 对日志消息中的敏感字段脱敏
        if hasattr(record, 'msg') and isinstance(record.msg, str):
            for key in self.SENSITIVE_KEYS:
                import re
                pattern = rf'({key}\s*[:=]\s*)([^\s,}}]+)'
                record.msg = re.sub(pattern, r'\1******', record.msg, flags=re.IGNORECASE)
        return True
```

```java
@Component
public class DynamicConfigRefresher {
    
    private static final Logger log = LoggerFactory.getLogger(DynamicConfigRefresher.class);
    
    // 支持热刷新的配置项
    private final AtomicInteger activeThreadCount = new AtomicInteger(8);
    private final AtomicLong testTimeout = new AtomicLong(300);
    private final AtomicBoolean gatedLaunch = new AtomicBoolean(false);
    
    @Scheduled(fixedDelay = 30000) // 每30秒检查一次
    public void refreshFromConfigCenter() {
        try {
            Config config = ConfigService.getAppConfig();
            
            // 线程数热变更
            int newThreadCount = config.getIntProperty("execution.thread.count", 8);
            int oldThreadCount = activeThreadCount.getAndSet(newThreadCount);
            if (newThreadCount != oldThreadCount) {
                log.info("线程池大小变更: {} -> {}", oldThreadCount, newThreadCount);
                resizeExecutorPool(newThreadCount);
            }
            
            // 超时时间热变更
            long newTimeout = config.getLongProperty("execution.timeout.seconds", 300);
            testTimeout.set(newTimeout);
            
            // 功能开关热变更
            boolean gateStatus = config.getBooleanProperty("feature.gated_launch", false);
            gatedLaunch.set(gateStatus);
            
            log.debug("配置热刷新完成");
        } catch (Exception e) {
            log.warn("配置刷新失败，保持上次配置: {}", e.getMessage());
        }
    }
    
    public int getActiveThreadCount() { return activeThreadCount.get(); }
    public long getTestTimeout() { return testTimeout.get(); }
}
```

### 案例4：配置校验与启动健康检查

**背景：** 框架启动时必须验证所有必要配置都已加载，不满足条件则阻止启动，避免运行到一半才发现配置缺失。

```java
@Component
public class ConfigHealthChecker implements ApplicationRunner {
    
    private static final Logger log = LoggerFactory.getLogger(ConfigHealthChecker.class);
    
    // 必需配置项清单
    private static final List<RequiredConfig> REQUIRED_CONFIGS = Arrays.asList(
        new RequiredConfig("api.base-url", "API基础地址"),
        new RequiredConfig("datasource.mysql.host", "MySQL主机"),
        new RequiredConfig("datasource.mysql.port", "MySQL端口"),
        new RequiredConfig("datasource.mysql.database", "MySQL数据库名"),
        new RequiredConfig("datasource.mysql.username", "MySQL用户名"),
        new RequiredConfig("report.path", "报告输出路径"),
        new RequiredConfig("execution.retry.max", "最大重试次数")
    );
    
    @Override
    public void run(ApplicationArguments args) {
        log.info("========== 配置健康检查开始 ==========");
        List<String> errors = new ArrayList<>();
        List<String> warnings = new ArrayList<>();
        
        for (RequiredConfig rc : REQUIRED_CONFIGS) {
            String value = environment.getProperty(rc.key);
            if (value == null || value.trim().isEmpty()) {
                errors.add(String.format("缺少必需配置: %s (%s)", rc.key, rc.description));
            } else {
                log.info("[PASS] {} = {}", rc.key, 
                    rc.key.contains("password") ? "******" : value);
            }
        }
        
        // 数据库连通性检查
        if (errors.isEmpty()) {
            try {
                checkDatabaseConnection();
                log.info("[PASS] 数据库连接正常");
            } catch (Exception e) {
                warnings.add("数据库连接失败: " + e.getMessage());
                log.warn("[WARN] 数据库连接失败（仅告警，不阻止启动）: {}", e.getMessage());
            }
        }
        
        // 磁盘空间检查
        String reportPath = environment.getProperty("report.path", "./reports");
        File reportDir = new File(reportPath);
        if (reportDir.exists() && reportDir.getFreeSpace() < 1024 * 1024 * 100) {
            warnings.add("报告目录剩余空间不足100MB");
        }
        
        if (!errors.isEmpty()) {
            log.error("========== 配置健康检查失败 ==========");
            errors.forEach(log::error);
            throw new FrameworkInitializationException("框架配置不满足启动条件");
        }
        
        warnings.forEach(log::warn);
        log.info("========== 配置健康检查通过 ==========");
    }
    
    @Data
    @AllArgsConstructor
    private static class RequiredConfig {
        private String key;
        private String description;
    }
}
```

**配置合规检查工具：**
```python
# tools/config_compliance_check.py
import yaml
import jsonschema
from pathlib import Path

# 配置标准JSON Schema
CONFIG_SCHEMA = {
    "type": "object",
    "required": ["metadata", "environments", "execution", "report"],
    "properties": {
        "metadata": {
            "type": "object",
            "required": ["project", "team", "maintainer", "version"],
        },
        "execution": {
            "type": "object",
            "properties": {
                "parallel": {
                    "type": "object",
                    "properties": {
                        "threads": {"type": "integer", "minimum": 1, "maximum": 64},
                    }
                },
                "retry": {
                    "type": "object",
                    "properties": {
                        "max": {"type": "integer", "minimum": 0, "maximum": 5},
                    }
                }
            }
        }
    }
}

def check_config_compliance(config_dir: str) -> dict:
    """检查所有项目配置是否符合标准"""
    results = {"pass": [], "fail": []}
    
    for config_file in Path(config_dir).rglob("test-config.yaml"):
        try:
            with open(config_file, 'r', encoding='utf-8') as f:
                config = yaml.safe_load(f)
            
            jsonschema.validate(config, CONFIG_SCHEMA)
            results["pass"].append(str(config_file))
        except (jsonschema.ValidationError, yaml.YAMLError) as e:
            results["fail"].append({"file": str(config_file), "error": str(e)})
    
    return results

if __name__ == "__main__":
    results = check_config_compliance("projects/")
    print(f"合规: {len(results['pass'])} 个项目")
    print(f"不合规: {len(results['fail'])} 个项目")
    for fail in results['fail']:
        print(f"  - {fail['file']}: {fail['error']}")
```

## 本章小结

框架配置层是自动化测试体系的基石，设计良好的配置层应具备以下特征：

| 特征 | 说明 | 实现方式 |
|------|------|----------|
| **环境隔离** | 任意环境一键切换 | 多Profile + 环境变量 |
| **安全可靠** | 敏感信息不泄露 | 加密存储 + Vault/KMS |
| **动态可调** | 运行时不重启改配置 | 配置中心热刷新 |
| **启动校验** | 配置缺失时快速失败 | 健康检查 + Schema校验 |
| **标准化** | 多项目配置规范统一 | 配置模板 + 合规检查 |

> 配置是框架的DNA——它决定了框架能做什么，不能做什么。好的配置设计让框架灵活且健壮，差的配置设计让框架变成"硬编码的脚本集合"。
