# 夹具层

> **夹具层**（Fixture / Test Data Layer）：自动化测试框架的数据基础设施层，负责测试数据的准备、管理、清理和维护。夹具层的设计直接决定了测试用例的可重复性、隔离性和维护成本——"垃圾进、垃圾出"，测试数据的质量就是测试结果的可信度。

## 1. 测试数据准备策略

### 通俗释义

测试数据准备就是"为测试搭台"——好比拍电影前需要布置场景、准备道具。测试用例运行前，需要准备好它依赖的一切数据：登录账号、商品库存、订单记录、优惠券等。好的数据准备策略让测试用例"开箱即用"，不需要手动去数据库里插数据。

### 详细解释

#### 四种主流数据准备策略

```
策略对比：

策略1：硬编码 (Hardcoded)
    用例中直接写数据
    ├── 优点：简单直接
    ├── 缺点：数据分散、不可复用、维护噩梦
    └── 适用：Demo/验证概念

策略2：外部数据文件 (External Files)
    Excel/CSV/YAML/JSON存储测试数据
    ├── 优点：数据与代码分离、可编辑、可版本管理
    ├── 缺点：文件多时管理复杂
    └── 适用：数据驱动测试

策略3：数据工厂 (Data Factory)
    代码动态生成测试数据
    ├── 优点：灵活、可复用、支持复杂数据关系
    ├── 缺点：需要额外编码
    └── 适用：企业级测试框架

策略4：数据快照 (Database Snapshot)
    使用数据库备份/快照恢复测试数据
    ├── 优点：数据真实、恢复快
    ├── 缺点：快照维护成本高、占用空间
    └── 适用：回归测试、性能测试
```

#### 策略选择决策树

```
                    ┌─ 测试数据类型简单？ ── 是 ──> 外部数据文件
                    │
开始 ──> 数据量大？ ─┤
                    │                         ┌─> 数据工厂
                    └─ 是 ──> 数据关系复杂？ ──┤
                                              └─> 数据快照
```

#### 数据准备的生命周期

```
测试套件启动
    │
    ├─ [Suite Setup] 准备共享数据（耗时操作在此完成）
    │     ├── 初始化测试账号
    │     ├── 准备基础数据（商品类目、配置等）
    │     └── 启动Mock服务
    │
    ├─ [Class Setup] 准备模块级数据
    │     ├── 创建测试用商品
    │     └── 初始化模块配置
    │
    ├─ [Method Setup] 准备用例级数据（每个用例独立）
    │     ├── 创建订单前提数据
    │     └── 设置Mock期望
    │
    ├─ [Test Execution] 执行测试
    │
    ├─ [Method Teardown] 清理用例级数据
    │     ├── 删除创建的数据
    │     └── 重置Mock状态
    │
    ├─ [Class Teardown] 清理模块级数据
    │
    └─ [Suite Teardown] 清理所有测试痕迹
```

## 2. 数据工厂模式

### 通俗释义

数据工厂就像"3D打印机"——你告诉它想要什么类型的测试数据（一个已支付的订单、一个带优惠券的用户），它自动生成所有相关联的数据（先创建用户、再创建商品、再创建订单、再创建支付记录），你完全不用关心数据是怎么"造"出来的。

### 详细解释

#### Java数据工厂实现

```java
// TestDataFactory.java - 数据工厂核心
@Repository
public class TestDataFactory {
    
    @Autowired private DatabaseHelper dbHelper;
    @Autowired private HttpClientManager httpClient;
    
    private static final Logger log = LoggerFactory.getLogger(TestDataFactory.class);
    
    // ===== 用户数据 =====
    public User createUser(UserTemplate template) {
        String userId = "U" + System.currentTimeMillis();
        String defaultPassword = CryptoHelper.md5("Test@123");
        
        User user = User.builder()
            .userId(userId)
            .username(template.getUsername() != null ? 
                template.getUsername() : "testuser_" + userId)
            .password(defaultPassword)
            .email(template.getEmail() != null ? 
                template.getEmail() : userId + "@test.com")
            .phone(template.getPhone() != null ?
                template.getPhone() : TestDataGenerator.randomPhoneNumber())
            .role(template.getRole() != null ? template.getRole() : "USER")
            .status(template.getStatus() != null ? template.getStatus() : "ACTIVE")
            .build();
        
        // 通过API创建（模拟真实用户注册流程）
        HttpResponse response = httpClient.post("/api/users/register", 
            JsonUtil.toJson(user));
        response.assertStatus(201);
        
        log.info("[DataFactory] 创建用户: {} ({})", user.getUsername(), user.getUserId());
        return user;
    }
    
    public User createDefaultUser() {
        return createUser(UserTemplate.defaultTemplate());
    }
    
    public User createVipUser() {
        return createUser(UserTemplate.builder()
            .role("VIP")
            .build());
    }
    
    // ===== 商品数据 =====
    public Product createProduct(ProductTemplate template) {
        String productId = "P" + System.currentTimeMillis();
        
        Product product = Product.builder()
            .productId(productId)
            .name(template.getName() != null ? 
                template.getName() : "测试商品_" + productId)
            .price(template.getPrice() != null ? template.getPrice() : 
                new BigDecimal("99.00"))
            .stock(template.getStock() != null ? template.getStock() : 1000)
            .category(template.getCategory() != null ? 
                template.getCategory() : "GENERAL")
            .status("ON_SALE")
            .build();
        
        // 通过数据库直接插入（跳过后端流程，仅需数据存在）
        dbHelper.update("default",
            "INSERT INTO products VALUES (?,?,?,?,?,?)",
            product.getProductId(), product.getName(), product.getPrice(),
            product.getStock(), product.getCategory(), product.getStatus()
        );
        
        log.info("[DataFactory] 创建商品: {} (库存:{})", 
            product.getName(), product.getStock());
        return product;
    }
    
    // ===== 订单数据 =====
    public Order createOrder(User user, Product product, int quantity) {
        // 通过API创建（走完整业务流程）
        httpClient.addDefaultHeader("Authorization", "Bearer " + user.getToken());
        
        String orderJson = String.format(
            """{"userId":"%s","items":[{"productId":"%s","quantity":%d}]}""",
            user.getUserId(), product.getProductId(), quantity
        );
        
        HttpResponse response = httpClient.post("/api/orders", orderJson);
        response.assertStatus(201);
        
        Order order = response.toObject(Order.class);
        log.info("[DataFactory] 创建订单: {} (用户:{}, 金额:{})", 
            order.getOrderId(), user.getUsername(), order.getTotalAmount());
        
        return order;
    }
    
    // ===== 已支付订单（组合场景） =====
    public Order createPaidOrder() {
        User user = createDefaultUser();
        Product product = createProduct(ProductTemplate.defaultTemplate());
        Order order = createOrder(user, product, 2);
        payOrder(order);
        return order;
    }
    
    private void payOrder(Order order) {
        HttpResponse response = httpClient.post("/api/payments",
            String.format("""{"orderId":"%s","method":"ALIPAY"}""", order.getOrderId()));
        response.assertStatus(200);
        log.info("[DataFactory] 支付订单: {}", order.getOrderId());
    }
    
    // ===== 批量创建 =====
    public List<User> createUsers(int count) {
        return IntStream.range(0, count)
            .mapToObj(i -> createDefaultUser())
            .collect(Collectors.toList());
    }
    
    public List<Order> createOrdersWithDifferentStatus(User user) {
        Product product = createProduct(ProductTemplate.defaultTemplate());
        return List.of(
            createOrder(user, product, 1),                                    // 待支付
            payOrder(createOrder(user, product, 2)),                          // 已支付
            cancelOrder(createOrder(user, product, 1)),                       // 已取消
            refundOrder(payOrder(createOrder(user, product, 3)))              // 已退款
        );
    }
}

// 数据模板（Builder模式）

    }
}
```

#### Python数据工厂实现

```python
# data_factory.py
import random
from typing import List, Optional
from dataclasses import dataclass, field
from datetime import datetime
import logging

logger = logging.getLogger(__name__)

@dataclass
class User:
    user_id: str
    username: str
    email: str
    phone: str
    role: str
    status: str
    token: str = ""

@dataclass
class Product:
    product_id: str
    name: str
    price: float
    stock: int
    status: str

@dataclass
class Order:
    order_id: str
    user_id: str
    total_amount: float
    status: str
    items: list = field(default_factory=list)

class DataFactory:
    """测试数据工厂"""
    
    def __init__(self, http_client, db_helper):
        self.api = http_client
        self.db = db_helper
        self._created_users: List[User] = []
        self._created_products: List[Product] = []
        self._created_orders: List[Order] = []
    
    def create_user(self, username: str = None, role: str = "USER",
                   status: str = "ACTIVE") -> User:
        """创建用户"""
        uid = f"U{int(datetime.now().timestamp() * 1000)}"
        username = username or f"testuser_{uid}"
        
        user_data = {
            "userId": uid,
            "username": username,
            "password": "Test@123",
            "email": f"{uid}@test.com",
            "phone": f"138{random.randint(10000000, 99999999)}",
            "role": role,
            "status": status
        }
        
        resp = self.api.post("/api/users/register", json=user_data)
        resp.assert_status(201)
        
        token = resp.json().get("token", "")
        user = User(**user_data)
        user.token = token
        
        self._created_users.append(user)
        logger.info(f"[DataFactory] 创建用户: {user.username} ({user.user_id})")
        return user
    
    def create_product(self, name: str = None, price: float = 99.0,
                      stock: int = 1000, category: str = "GENERAL") -> Product:
        """创建商品"""
        pid = f"P{int(datetime.now().timestamp() * 1000)}"
        name = name or f"测试商品_{pid}"
        
        self.db.execute("default",
            "INSERT INTO products VALUES (?,?,?,?,?)",
            (pid, name, price, stock, category)
        )
        
        product = Product(pid, name, price, stock, "ON_SALE")
        self._created_products.append(product)
        logger.info(f"[DataFactory] 创建商品: {product.name} (库存:{stock})")
        return product
    
    def create_order(self, user: User, product: Product, 
                    quantity: int = 1) -> Order:
        """创建订单"""
        self.api.set_auth_token(user.token)
        
        order_data = {
            "userId": user.user_id,
            "items": [{"productId": product.product_id, "quantity": quantity}]
        }
        
        resp = self.api.post("/api/orders", json=order_data)
        resp.assert_status(201)
        
        order = Order(**resp.json())
        self._created_orders.append(order)
        logger.info(f"[DataFactory] 创建订单: {order.order_id}")
        return order
    
    def create_paid_order(self) -> Order:
        """创建已支付订单（组合操作）"""
        user = self.create_user()
        product = self.create_product()
        order = self.create_order(user, product, 2)
        
        self.api.set_auth_token(user.token)
        resp = self.api.post("/api/payments",
            json={"orderId": order.order_id, "method": "ALIPAY"})
        resp.assert_status(200)
        order.status = "PAID"
        
        logger.info(f"[DataFactory] 创建已支付订单: {order.order_id}")
        return order
    
    def cleanup(self):
        """清理所有创建的测试数据"""
        logger.info("[DataFactory] 开始清理测试数据...")
        
        # 按依赖顺序清理（先子后父）
        for order in reversed(self._created_orders):
            try:
                self.db.execute("default",
                    "DELETE FROM orders WHERE order_id = ?", (order.order_id,))
            except Exception as e:
                logger.warning(f"清理订单失败 {order.order_id}: {e}")
        
        for product in self._created_products:
            try:
                self.db.execute("default",
                    "DELETE FROM products WHERE product_id = ?", 
                    (product.product_id,))
            except Exception as e:
                logger.warning(f"清理商品失败 {product.product_id}: {e}")
        
        for user in self._created_users:
            try:
                self.db.execute("default",
                    "DELETE FROM users WHERE user_id = ?", (user.user_id,))
            except Exception as e:
                logger.warning(f"清理用户失败 {user.user_id}: {e}")
        
        self._created_users.clear()
        self._created_products.clear()
        self._created_orders.clear()
        logger.info("[DataFactory] 测试数据清理完成")
```

## 3. 数据清理策略

### 通俗释义

数据清理就像是"用完餐后收拾桌子"——测试跑完后必须清理产生的数据，不然测试环境会被垃圾数据填满，下一次测试可能因为这些残留数据而失败。

### 详细解释

#### 四种清理策略对比

| 策略 | 机制 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| **标记清理** | 给测试数据打标记（如 `source='test'`），批量删除 | 简单、批量操作快 | 侵入业务表（增加字段） | 可控制数据源的场景 |
| **时间戳清理** | 记录创建时间，定时清理超过一定时间的数据 | 实现简单 | 延迟清理、可能漏删 | 测试环境数据清理 |
| **事务回滚** | 每个用例包裹在数据库事务中，执行完回滚 | 彻底、无残留 | 不支持跨事务操作 | 单库操作、单元测试 |
| **快照恢复** | 数据库恢复到测试前的快照 | 最彻底 | 恢复耗时、需要快照 | 回归测试 |

#### Java事务回滚方案

```java
// TransactionalTestBase.java
public abstract class TransactionalTestBase {
    
    @Autowired protected DataSource dataSource;
    @Autowired protected JdbcTemplate jdbcTemplate;
    
    private TransactionStatus transactionStatus;
    private PlatformTransactionManager txManager;
    
    @BeforeMethod
    public void beginTransaction() {
        txManager = new DataSourceTransactionManager(dataSource);
        DefaultTransactionDefinition def = new DefaultTransactionDefinition();
        def.setPropagationBehavior(TransactionDefinition.PROPAGATION_REQUIRES_NEW);
        transactionStatus = txManager.getTransaction(def);
        
        log.info("[TX] 事务开始");
    }
    
    @AfterMethod
    public void rollbackTransaction() {
        if (transactionStatus != null && !transactionStatus.isCompleted()) {
            txManager.rollback(transactionStatus);
            log.info("[TX] 事务回滚（测试数据已清除）");
        }
    }
}
```

#### Python标记清理方案

```python
# data_cleaner.py
import uuid
from datetime import datetime, timedelta
from contextlib import contextmanager

class TestDataCleaner:
    """测试数据清理器"""
    
    def __init__(self, db_helper):
        self.db = db_helper
        self.ds = "default"
        # 生成唯一的测试批次标识
        self.batch_id = str(uuid.uuid4())[:8]
        self.marked_tables = {}
    
    def mark_for_cleanup(self, table: str, record_id: str):
        """标记需要清理的记录"""
        if table not in self.marked_tables:
            self.marked_tables[table] = []
        self.marked_tables[table].append(record_id)
    
    def insert_with_mark(self, table: str, data: dict, 
                        mark_column: str = "test_batch") -> int:
        """插入数据并自动标记（确保表有test_batch列）"""
        data[mark_column] = self.batch_id
        columns = ', '.join(data.keys())
        placeholders = ', '.join(['%s'] * len(data))
        sql = f"INSERT INTO {table} ({columns}) VALUES ({placeholders})"
        
        affected = self.db.execute(self.ds, sql, tuple(data.values()))
        self.mark_for_cleanup(table, str(data.get('id', '')))
        return affected
    
    def cleanup_batch(self):
        """按批次清理"""
        for table, ids in self.marked_tables.items():
            if ids:
                placeholders = ', '.join(['%s'] * len(ids))
                self.db.execute(self.ds,
                    f"DELETE FROM {table} WHERE id IN ({placeholders})",
                    tuple(ids))
        
        self.marked_tables.clear()
    
    def cleanup_by_time(self, table: str, hours: int = 24):
        """按时间清理（假设有created_at列）"""
        cutoff = (datetime.now() - timedelta(hours=hours)).isoformat()
        affected = self.db.execute(self.ds,
            f"DELETE FROM {table} WHERE test_batch IS NOT NULL "
            f"AND created_at < %s", (cutoff,))
        return affected
    
    @contextmanager
    def auto_cleanup(self):
        """上下文管理器：自动清理"""
        try:
            yield self
        finally:
            self.cleanup_batch()


# 使用示例
cleaner = TestDataCleaner(db_helper)
with cleaner.auto_cleanup():
    cleaner.insert_with_mark("users", {"name": "testuser", "email": "test@test.com"})
    # 测试逻辑...
    # 自动清理
```

## 4. 数据库快照与回滚

### 通俗释义

数据库快照就像是"游戏存点"——在测试开始前给数据库拍一张"快照"，测试跑完后直接恢复到快照状态，就好像从来没跑过测试一样。这是最彻底的清理方式。

### 详细解释

```java
// DatabaseSnapshotManager.java
public class DatabaseSnapshotManager {
    
    private final DataSource dataSource;
    private final JdbcTemplate jdbcTemplate;
    private static final Logger log = LoggerFactory.getLogger(DatabaseSnapshotManager.class);
    
    // MySQL快照操作
    public void createSnapshot(String snapshotName) {
        String snapshotTable = "test_snapshot_" + snapshotName;
        
        // 复制关键表到快照表
        String[] criticalTables = {"users", "orders", "products", "payments", "inventory"};
        for (String table : criticalTables) {
            jdbcTemplate.execute(String.format(
                "CREATE TABLE %s_%s AS SELECT * FROM %s",
                snapshotTable, table, table
            ));
            int count = jdbcTemplate.queryForObject(
                "SELECT COUNT(*) FROM " + table, Integer.class);
            log.info("[Snapshot] {} : {} 条记录已备份", table, count);
        }
        
        log.info("[Snapshot] 快照完成: {}", snapshotName);
    }
    
    public void restoreSnapshot(String snapshotName) {
        String snapshotTable = "test_snapshot_" + snapshotName;
        String[] criticalTables = {"users", "orders", "products", "payments", "inventory"};
        
        for (String table : criticalTables) {
            jdbcTemplate.execute(String.format("TRUNCATE TABLE %s", table));
            jdbcTemplate.execute(String.format(
                "INSERT INTO %s SELECT * FROM %s_%s", table, snapshotTable, table));
            jdbcTemplate.execute(String.format(
                "DROP TABLE IF EXISTS %s_%s", snapshotTable, table));
        }
        
        log.info("[Snapshot] 快照恢复完成: {}", snapshotName);
    }
    
    // Docker容器级快照（更彻底）
    public void dockerSnapshot(String containerName, String snapshotName) {
        try {
            // 提交容器为新镜像
            ProcessBuilder pb = new ProcessBuilder(
                "docker", "commit", containerName, 
                "mysql-snapshot:" + snapshotName
            );
            Process process = pb.start();
            process.waitFor();
            log.info("[DockerSnapshot] 容器快照创建: {}", snapshotName);
        } catch (Exception e) {
            throw new RuntimeException("Docker快照失败", e);
        }
    }
    
    public void dockerRestore(String containerName, String snapshotName) {
        try {
            // 停止容器
            new ProcessBuilder("docker", "stop", containerName).start().waitFor();
            // 删除容器
            new ProcessBuilder("docker", "rm", containerName).start().waitFor();
            // 从快照镜像启动新容器
            new ProcessBuilder("docker", "run", "-d", "--name", containerName,
                "-p", "3306:3306", "mysql-snapshot:" + snapshotName).start().waitFor();
            // 等待MySQL启动
            Thread.sleep(5000);
            log.info("[DockerSnapshot] 容器恢复完成: {}", snapshotName);
        } catch (Exception e) {
            throw new RuntimeException("Docker恢复失败", e);
        }
    }
}
```

## 5. 共享夹具 vs 独立夹具

### 通俗释义

共享夹具好比"自助餐厅的公共餐具"——大家都能用，但用的时候要小心别弄脏。独立夹具好比"一次性餐具"——每个人用自己那份，用完就扔，干净卫生但成本高。

### 详细解释

| 维度 | 共享夹具 (Shared Fixture) | 独立夹具 (Isolated Fixture) |
|------|---------------------------|------------------------------|
| **数据创建** | 套件开始时创建一次 | 每个用例创建自己的 |
| **执行速度** | 快（只创建一次） | 慢（每次都要创建） |
| **用例隔离** | 差（一个用例可能破坏共享数据） | 好（完全隔离） |
| **维护成本** | 高（需确保数据不被破坏） | 低（各管各的） |
| **适用场景** | 只读数据（商品类目、配置） | 可变数据（订单、用户） |
| **并发安全** | 需要保护 | 天然安全 |

#### 选择原则

```
原则1：只读数据用共享，可变数据用独立
    ├── 共享：商品类目、地区列表、系统配置
    └── 独立：用户账户、订单、支付记录

原则2：创建成本高的用共享，创建成本低的用独立
    ├── 共享：需要多个依赖的大业务对象
    └── 独立：简单的单表数据

原则3：串行执行可共享，并发执行需独立
    ├── 共享：串行执行的端到端流程
    └── 独立：高并发的API测试
```

```python
# testdata_version_manager.py
import yaml
import json
from pathlib import Path
from datetime import datetime
from typing import Dict, List
import hashlib

class TestDataVersionManager:
    """测试数据版本管理器"""
    
    def __init__(self, data_dir: str = "testdata"):
        self.data_dir = Path(data_dir)
        self.version_file = self.data_dir / "VERSION"
        self.changelog_file = self.data_dir / "CHANGELOG.md"
    
    @property
    def current_version(self) -> str:
        """获取当前数据版本"""
        if self.version_file.exists():
            return self.version_file.read_text().strip()
        return "0.0.0"
    
    def bump_version(self, level: str = "patch"):
        """升级版本号（major/minor/patch）"""
        major, minor, patch = map(int, self.current_version.split('.'))
        
        if level == "major":
            major += 1; minor = 0; patch = 0
        elif level == "minor":
            minor += 1; patch = 0
        else:
            patch += 1
        
        new_version = f"{major}.{minor}.{patch}"
        self.version_file.write_text(new_version)
        return new_version
    
    def add_changelog(self, version: str, changes: List[str]):
        """添加变更记录"""
        date_str = datetime.now().strftime("%Y-%m-%d")
        entry = f"\n## {version} ({date_str})\n\n"
        for change in changes:
            entry += f"- {change}\n"
        
        with open(self.changelog_file, 'a', encoding='utf-8') as f:
            f.write(entry)
    
    def compute_data_hash(self) -> Dict[str, str]:
        """计算测试数据的哈希值（用于检测变更）"""
        hashes = {}
        for data_file in sorted(self.data_dir.glob("v*/**/*")):
            if data_file.is_file() and data_file.suffix in ('.yaml', '.json', '.csv'):
                content = data_file.read_bytes()
                hashes[str(data_file.relative_to(self.data_dir))] = \
                    hashlib.sha256(content).hexdigest()[:16]
        return hashes
    
    def validate_data_schema(self, schema_dir: str = "schemas") -> List[str]:
        """验证测试数据是否符合Schema"""
        errors = []
        from jsonschema import validate, ValidationError
        
        for data_file in self.data_dir.glob("v*/**/*.yaml"):
            schema_file = Path(schema_dir) / f"{data_file.stem}_schema.yaml"
            if schema_file.exists():
                with open(data_file) as df, open(schema_file) as sf:
                    try:
                        validate(yaml.safe_load(df), yaml.safe_load(sf))
                    except ValidationError as e:
                        errors.append(f"{data_file.name}: {e.message}")
        
        return errors
    
    def check_data_consistency(self) -> List[str]:
        """检查测试数据的一致性"""
        warnings = []
        
        # 检查是否有引用不存在的数据
        all_user_ids = set()
        all_product_ids = set()
        
        for user_file in (self.data_dir / f"v{self.current_version}" / "users").glob("*.yaml"):
            user = yaml.safe_load(user_file.read_text())
            all_user_ids.add(user.get("userId"))
        
        for order_file in (self.data_dir / f"v{self.current_version}" / "orders").glob("*.json"):
            order = json.loads(order_file.read_text())
            if order.get("userId") not in all_user_ids:
                warnings.append(f"订单 {order.get('orderId')} 引用了不存在的用户")
        
        return warnings
```

```python
# data_warmup.py
import pytest
from concurrent.futures import ThreadPoolExecutor
import time

class DataWarmUp:
    """测试数据预热器——提前批量创建高频使用的测试数据"""
    
    def __init__(self, data_factory, pool_size: int = 50):
        self.factory = data_factory
        self.pool_size = pool_size
        self._warmed_users: list = []
        self._warmed_products: list = []
    
    def warm_up(self):
        """预热数据池"""
        start = time.time()
        
        with ThreadPoolExecutor(max_workers=8) as executor:
            # 预创建用户
            user_futures = [
                executor.submit(self.factory.create_user) 
                for _ in range(self.pool_size)
            ]
            self._warmed_users = [f.result() for f in user_futures]
            
            # 预创建商品
            product_futures = [
                executor.submit(self.factory.create_product)
                for _ in range(self.pool_size)
            ]
            self._warmed_products = [f.result() for f in product_futures]
        
        elapsed = time.time() - start
        print(f"数据预热完成: {self.pool_size}用户 + {self.pool_size}商品 ({elapsed:.1f}s)")
    
    def get_user(self) -> object:
        """从预热池获取用户（消费后自动补充）"""
        if not self._warmed_users:
            self._warmed_users.append(self.factory.create_user())
        return self._warmed_users.pop()
    
    def get_product(self) -> object:
        if not self._warmed_products:
            self._warmed_products.append(self.factory.create_product())
        return self._warmed_products.pop()
    
    def cleanup(self):
        self.factory.cleanup_all_users(self._warmed_users)
        self.factory.cleanup_all_products(self._warmed_products)
```

```python
# data_consistency_checker.py
class DataConsistencyChecker:
    """数据库与缓存一致性校验"""
    
    def __init__(self, db_helper, cache_helper):
        self.db = db_helper
        self.cache = cache_helper
    
    def assert_db_cache_equals(self, table: str, key_field: str, 
                               key_value: str, fields: List[str]):
        """断言数据库与缓存数据一致"""
        # 从数据库查询
        db_record = self.db.query_one("default",
            f"SELECT {', '.join(fields)} FROM {table} WHERE {key_field} = %s",
            (key_value,))
        
        # 从缓存查询
        cache_key = f"{table}:{key_value}"
        cache_record = self.cache.get_json(cache_key)
        
        assert db_record is not None, f"数据库记录不存在: {table}.{key_field}={key_value}"
        assert cache_record is not None, f"缓存记录不存在: {cache_key}"
        
        mismatches = []
        for field in fields:
            db_val = db_record.get(field)
            cache_val = cache_record.get(field)
            # 类型统一比较
            if str(db_val) != str(cache_val):
                mismatches.append(f"  {field}: DB={db_val} Cache={cache_val}")
        
        assert not mismatches, \
            f"DB与缓存不一致 ({table}.{key_field}={key_value}):\n" + "\n".join(mismatches)
    
    def assert_cache_invalidated(self, key_pattern: str, within_seconds: int = 5):
        """断言缓存在指定时间内被清除"""
        import time
        deadline = time.time() + within_seconds
        
        while time.time() < deadline:
            keys = self.cache.keys(key_pattern)
            if not keys:
                return  # 缓存已清除
            time.sleep(0.5)
        
        remaining = self.cache.keys(key_pattern)
        assert not remaining, f"缓存未被清除: {remaining}"
```

```java
    }
    
    /**

     */
    public Order orchestrateWithRollback() {
        List<AutoCloseable> resources = new ArrayList<>();

            User user = createUserViaUserService();
            resources.add(() -> deleteUser(user.getUserId()));
```

## 本章小结

| 维度 | 最佳实践 | 反面教训 |
|------|----------|----------|
| 数据准备 | 使用数据工厂统一管理，代码与数据分离 | 在用例中硬编码SQL，数据散落各处 |
| 数据工厂 | Builder模式 + 模板方法，链式创建复杂对象 | 一个工厂方法堆砌所有参数 |
| 数据清理 | 标记清理 + 定时清理 + 事务回滚组合使用 | 不清理数据，测试环境逐渐腐烂 |
| 快照恢复 | 关键回归测试使用快照，快速可靠 | 每次测试都做全量快照，效率低下 |
| 夹具共享 | 只读共享，可变独立；创建成本决定策略 | 所有数据都共享导致用例相互污染 |
| 版本管理 | 测试数据跟随接口版本，有变更记录 | 接口升级后测试数据不更新导致假失败 |

> 夹具层的核心价值：让测试用例"拿到就能跑，跑完就干净"。好的夹具设计让测试用例开发者只需关心"测什么"，而不需关心"数据从哪来、用完怎么清"。
